Create `.claude/skills/api-upgrade/SKILL.md`. The file must open with a YAML frontmatter
block (triple-dash delimiters) with fields `name: api-upgrade` and a description string.

**## Purpose** — 3-4 sentences: mechanical version bump plus guided workflow consuming api-diff output.

**## When to Use**
- New FlashBlade firmware available with a new REST API version
- After api-diff has produced a migration plan (migration-plan.md)
- When ROADMAP.md needs updating for new/deprecated endpoints

**## Prerequisites**
- api-diff skill completed — `/tmp/migration-plan.md` (or equivalent) available
- `.claude/skills/api-upgrade/scripts/upgrade_version.py` present
- Python 3.10+ available

**## Workflow — Phase 1: Infrastructure**

Run the mechanical version bump:

    python3 .claude/skills/api-upgrade/scripts/upgrade_version.py \
      --from OLD --to NEW --apply

Then verify:

    make build
    make test

Commit: `chore: bump API version from OLD to NEW`.

**Review Gate 1** (#### heading): checkbox list:
- make build passes with zero errors
- make test passes (count >= previous baseline)
- const APIVersion in client.go shows NEW version
- Mock server versions slice contains NEW version
- All mock handler paths use /api/NEW/

Instruction line: "Type 'gate-1 passed' to continue."

**## Workflow — Phase 2: Schema Updates**

Consume `update_models` items from migration plan:
- For each modified endpoint: update Post/Patch/Get structs in `internal/client/models_<domain>.go`
- Add new fields with correct pointer rules (PATCH fields are pointers, GET fields are values)
- Run `make build` + `make test` after each model change

**Review Gate 2**: checkbox list:
- All update_models items from migration plan applied
- No compilation errors
- Drift detection logging added for new fields in Read methods
- make test passes

Instruction: "Type 'gate-2 passed' to continue."

**## Workflow — Phase 3: New Resources**

Consume `new_resources` items from migration plan. For each new endpoint, use the
`flashblade-resource-builder` skill (`.claude/skills/flashblade-resource-builder/SKILL.md`):
- Full lifecycle: models → client → mock handler → resource → data source → tests → examples
- Register in `internal/provider/provider.go`
- Run `make test` after each new resource

**Review Gate 3**: checkbox list:
- All new_resources items implemented
- Each new resource has >= 8 tests (4 client + 3 resource + 1 data source)
- make test passes, count increased by >= 8 per new resource
- All new resources registered in provider.go

Instruction: "Type 'gate-3 passed' to continue."

**## Workflow — Phase 4: Deprecations**

Consume `deprecated` items from migration plan:
- Remove client CRUD methods for fully removed endpoints
- Remove or archive resource/data source files
- Remove registration from provider.go
- Run `make build` + `make test`

Note: Stubs acceptable if resource is removed from API but still used — leave stub returning
AddError with deprecation message.

**Review Gate 4**: checkbox list:
- All deprecated items handled (removed or stubbed)
- No dead code referencing removed endpoints
- make build passes
- make test passes

Instruction: "Type 'gate-4 passed' to continue."

**## Workflow — Phase 5: Documentation**

Regenerate all documentation:

    PYTHONPATH=.claude/skills python3 .claude/skills/swagger-to-reference/scripts/parse_swagger.py \
      swagger-NEW.json --output api_references/NEW.md

    make docs

Then update FLASHBLADE_API.md if hand-curated sections changed, and update ROADMAP.md
to move new resources from Candidate to Implemented.

Reference: `.claude/skills/swagger-to-reference/SKILL.md`

Commit: `docs: update API reference and provider docs for vNEW`.

**Review Gate 5 (Final)**: checkbox list:
- api_references/NEW.md generated
- make docs regenerated all docs/ files
- ROADMAP.md updated: new resources moved from Candidate to Implemented
- FLASHBLADE_API.md updated if needed
- All commits pushed, PR opened

Instruction: "Type 'gate-5 passed' — upgrade complete."

**## Output**
- Updated provider targeting NEW API version
- `api_references/NEW.md` — AI-optimized reference for the new version
- ROADMAP.md updated with new resource coverage

**## Troubleshooting**

Table with 4 rows:
| Problem | Cause | Fix |

- make test count decreased after Phase 3 | New resource missing test coverage | Add missing tests per CONVENTIONS.md minimums
- make docs fails | Schema change without schema version bump | Increment SchemaVersion + add UpgradeState entry
- ImportError in parse_swagger.py | Missing PYTHONPATH | Prefix: PYTHONPATH=.claude/skills python3 ...
