**Step 1 â€” Extend DiagnosticReporter interface in helpers.go:**
Add `AddWarning(string, string)` to the DiagnosticReporter interface:
```go
type DiagnosticReporter interface {
    AddError(string, string)
    AddWarning(string, string)
    HasError() bool
}
```
All existing implementations are `*diag.Diagnostics` (passed as `&resp.Diagnostics`) â€” that type already has AddWarning, so this change is backward compatible.

**Step 2 â€” Fix readIntoState in filesystem_resource.go (lines ~663-666):**
Replace the current loop that promotes all diagnostics to errors:
```go
for _, d := range mapFSToModel(fs, data) {
    reporter.AddError(d.Summary(), d.Detail())
}
```
With severity-preserving dispatch:
```go
for _, d := range mapFSToModel(fs, data) {
    if d.Severity() == diag.SeverityWarning {
        reporter.AddWarning(d.Summary(), d.Detail())
    } else {
        reporter.AddError(d.Summary(), d.Detail())
    }
}
```
The `diag` package is already imported in filesystem_resource.go â€” no new imports needed.

**Verification of interface compliance:**
The `resp.Diagnostics` type is `diag.Diagnostics` which has both `AddError(string, string)` and `AddWarning(string, string)` methods.
