<objective>
Extend flashblade_object_store_remote_credentials to accept either a target reference OR an array connection reference (not both simultaneously), and verify that the bucket replica link resource works end-to-end when the remote credentials reference a target.

Purpose: RC-01 enables the S3 target replication workflow by allowing credentials to be scoped to a target instead of an array connection. RC-02 ensures array-connection credentials continue to work unchanged. BRL-01 validates the end-to-end replication path with target-backed credentials.

Output:
- Updated client.PostRemoteCredentials (new targetName param, conditional query routing)
- Schema v1 with target_name attribute + remote_name made Optional+Computed + state upgrader v0->v1
- Mock handler accepting ?target_names= or ?remote_names= on POST
- New unit tests covering target-based create, migration, and BRL-with-target-credentials scenario
</objective>
