1. Create `internal/provider/object_store_virtual_host_resource.go`:

- TypeName: `flashblade_object_store_virtual_host`
- Model struct `objectStoreVirtualHostModel`:
  - ID: types.String, computed, UseStateForUnknown
  - Name: types.String, computed, UseStateForUnknown — server-assigned, read-only. Used for PATCH/DELETE/GET queries. NOT user-supplied.
  - Hostname: types.String, required — the primary user-supplied field. Used for POST and writable via PATCH.
  - AttachedServers: types.List of types.StringType (server names as strings), optional, computed. Default empty list.
  - Timeouts: timeouts.Value
- Factory: `NewObjectStoreVirtualHostResource()`
- Create: Extract hostname and attached_servers from plan. Build ObjectStoreVirtualHostPost with hostname and NamedReference slice from string list. Call PostObjectStoreVirtualHost(ctx, hostname, body). Read server-assigned `name` from response. Store in state.
- Read: Use `data.Name` (server-assigned) for GetObjectStoreVirtualHost. 404->RemoveResource. Map response to model. Drift detection on hostname and attached_servers.
- Update: Compare plan vs state for hostname and attached_servers. Build ObjectStoreVirtualHostPatch. For attached_servers, always send the full desired list (full-replace semantics). Call PatchObjectStoreVirtualHost(ctx, name, patch). Read back state.
- Delete: Call DeleteObjectStoreVirtualHost(ctx, name). 404 is idempotent.
- Import: by `name` (server-assigned). Set Name in state, call readIntoState.
- Helpers: `mapVirtualHostToModel` — converts client.ObjectStoreVirtualHost to model. AttachedServers converts []NamedReference to types.List of server name strings. `readIntoState` — calls Get and maps.

CRITICAL: `name` is Computed (server-assigned, read-only). `hostname` is Required (user-supplied, writable). Import uses `name`, not `hostname`. PATCH/DELETE use `name` as query param.

2. Create `internal/provider/object_store_virtual_host_data_source.go`:
- TypeName: `flashblade_object_store_virtual_host`
- Schema: name(optional), filter(optional), id(computed), hostname(computed), attached_servers(computed, list of string), timeouts(read)
- Read: if name provided, call GetObjectStoreVirtualHost; if filter, call ListObjectStoreVirtualHosts

3. Register in `internal/provider/provider.go`:
- Add `NewObjectStoreVirtualHostResource` to `Resources()` list
- Add `NewObjectStoreVirtualHostDataSource` to `DataSources()` list

NOTE: provider.go is also modified by Plan 02. Since both plans add distinct entries to the Resources/DataSources slices, the executor should read the current state of provider.go before modifying.
