1. Create `internal/provider/s3_export_policy_resource.go` by adapting `nfs_export_policy_resource.go`:

- TypeName: `flashblade_s3_export_policy`
- Model struct `s3ExportPolicyModel` with fields: ID(computed), Name(required), Enabled(optional+computed, default true), IsLocal(computed+UseStateForUnknown), PolicyType(computed+UseStateForUnknown), Version(computed), Timeouts
- Factory: `NewS3ExportPolicyResource()`
- CRUD: Create calls `PostS3ExportPolicy`, Read calls `GetS3ExportPolicy` with 404->RemoveResource, Update calls `PatchS3ExportPolicy` (supports rename via old name + new name), Delete calls `DeleteS3ExportPolicy` (NO member guard — S3 policies don't have file system members)
- Import: by name, same pattern as NFS
- Helper: `mapS3PolicyToModel`, `readIntoState`
- Drift detection on `enabled` field

2. Create `internal/provider/s3_export_policy_rule_resource.go` by adapting `nfs_export_policy_rule_resource.go`:

- TypeName: `flashblade_s3_export_policy_rule`
- Model struct `s3ExportPolicyRuleModel` with fields: ID(computed+UseStateForUnknown), PolicyName(required+RequiresReplace), Name(computed+UseStateForUnknown — server-assigned), Index(computed), Effect(required — NO RequiresReplace, effect IS patchable unlike OAP), Actions(required, types.List of string), Resources(required, types.List of string), Timeouts
- Factory: `NewS3ExportPolicyRuleResource()`
- Create: build S3ExportPolicyRulePost from plan (effect, actions, resources), call PostS3ExportPolicyRule, read back by server-assigned name
- Read: call GetS3ExportPolicyRuleByName, 404->RemoveResource, drift detection on effect
- Update: compare plan vs state for effect/actions/resources, build S3ExportPolicyRulePatch, call PatchS3ExportPolicyRule. For effect use pointer (`*string`). For actions/resources use slice comparison.
- Delete: call DeleteS3ExportPolicyRule with policyName and ruleName
- Import: composite ID `policy_name/rule_index` — parse with SplitN("/", 2), parse index as int, call GetS3ExportPolicyRuleByIndex, populate state
- Helper: `mapS3RuleToModel` — maps client.S3ExportPolicyRule to model, converts actions/resources to types.List

CRITICAL DIFFERENCES from NFS rule and OAP rule:
- Effect is NOT RequiresReplace (patchable via PATCH body)
- Name is server-assigned (NOT user-supplied) — Computed + UseStateForUnknown
- No `conditions` field (unlike OAP)
- Import uses `policy_name/rule_index` (like NFS, NOT `policy_name/rule_name` like OAP)

3. Create `internal/provider/s3_export_policy_data_source.go` following the NFS export policy data source pattern:
- TypeName: `flashblade_s3_export_policy`
- Schema: name(optional), filter(optional), id(computed), enabled(computed), is_local(computed), policy_type(computed), version(computed), timeouts(read)
- Read: if name provided, call GetS3ExportPolicy; if filter, call ListS3ExportPolicies with filter

4. Register in `internal/provider/provider.go`:
- Add `NewS3ExportPolicyResource` and `NewS3ExportPolicyRuleResource` to `Resources()` list
- Add `NewS3ExportPolicyDataSource` to `DataSources()` list
