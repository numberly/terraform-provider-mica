**object_store_access_policy_resource.go** — Follow nfs_export_policy_resource.go pattern:

Schema attributes:
- `id`: Computed, UseStateForUnknown
- `name`: Required (writable — rename in-place via PATCH)
- `description`: Optional, Computed, RequiresReplace — POST-only field, cannot be updated (per research pitfall 3)
- `arn`: Computed, UseStateForUnknown
- `enabled`: Computed (read-only on this API — not writable via PATCH)
- `is_local`: Computed, UseStateForUnknown
- `policy_type`: Computed, UseStateForUnknown
- `timeouts`: standard Create/Read/Update/Delete

CRUD methods:
- Create: POST /object-store-access-policies?names= with body {description}. Read back state.
- Read: GET by name. Drift detection on name, description.
- Update: PATCH by OLD name. Only `name` is patchable. Build patch comparing plan vs state.
- Delete: Delete guard — check ListObjectStoreAccessPolicyMembers. If members > 0, return diagnostic error. Else DELETE.
- ImportState: By name. Initialize timeouts null object. Set Name, call readIntoState.

Helper functions:
- `readIntoState(ctx, name, *model, diags)` — same pattern as NFS
- `mapOAPToModel(policy, *model)` — map client struct to TF model

**object_store_access_policy_rule_resource.go** — Follow smb_share_policy_rule_resource.go pattern (name-based import, not index-based):

Schema attributes:
- `id`: Computed (this will be synthetic "policyName/ruleName")
- `policy_name`: Required, RequiresReplace
- `name`: Required — the user provides the rule name (used in POST ?names= query param). RequiresReplace (cannot rename rules).
- `effect`: Required, RequiresReplace — "allow" or "deny", READ-ONLY after creation (research pitfall 2). Add stringvalidator.OneOf("allow", "deny").
- `actions`: Required, List of strings — e.g. ["s3:GetObject", "s3:PutObject"]
- `resources`: Required, List of strings — ARN-like resource patterns
- `conditions`: Optional, String — JSON string via jsonencode(). Stored as types.String. When reading back from API, marshal json.RawMessage to string. Use jsontypes or plain types.String.
- `timeouts`: standard

CRUD methods:
- Create: POST with policyName and ruleName as query params. Body has effect, actions, conditions (parse string to json.RawMessage), resources. Read back.
- Read: GET by policyName+ruleName. Map json.RawMessage conditions back to types.String. Drift detection on actions.
- Update: PATCH by policyName+ruleName. Only actions, resources, conditions are patchable (effect is RequiresReplace). Convert conditions string to json.RawMessage for PATCH body.
- Delete: DELETE by policyName+ruleName.
- ImportState: Parse "policy_name/rule_name" composite ID (use SplitN "/" 2). Look up by name.

Helper: `mapOAPRuleToModel(ctx, rule, *model)` — handle conditions json.RawMessage to types.String conversion. If conditions is nil or empty `{}`, set to types.StringNull(). Otherwise string(rule.Conditions).
