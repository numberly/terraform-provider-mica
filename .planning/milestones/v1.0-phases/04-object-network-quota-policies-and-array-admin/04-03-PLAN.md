**network_access_policy_resource.go** — SINGLETON pattern (NOT standard CRUD):

Schema attributes:
- `id`: Computed, UseStateForUnknown
- `name`: Required — the policy name to manage (system-provided policies exist, user references by name)
- `enabled`: Optional, Computed, Default true — writable via PATCH
- `is_local`: Computed, UseStateForUnknown
- `policy_type`: Computed, UseStateForUnknown
- `version`: Computed
- `timeouts`: standard Create/Read/Update/Delete

CRUD methods — SINGLETON LIFECYCLE:
- **Create**: This is NOT a POST. Steps: (1) GET /network-access-policies?names= to verify the policy exists and read current state. If not found, return error diagnostic "Network access policy not found — NAP policies are system-managed and cannot be created". (2) PATCH with desired config (enabled). (3) Read back full state.
- **Read**: GET by name. Standard drift detection on `enabled`. If not found, RemoveResource.
- **Update**: PATCH by old name. Build patch comparing plan vs state for enabled and name. After rename, use new name for readback.
- **Delete**: PATCH to reset — set enabled=false. This is the "destroy" for a singleton: return to default/disabled state. Do NOT call any DELETE endpoint (none exists). Log via tflog that NAP policy was reset to disabled state.
- **ImportState**: By name (not "default" — NAP may have multiple named policies on a FlashBlade, per research OQ-3). Initialize timeouts null, set Name, readIntoState.

Helpers:
- `readIntoState(ctx, name, *model, diags)` — GET and map
- `mapNAPToModel(policy, *model)` — map client struct to TF model

**network_access_policy_rule_resource.go** — Follow nfs_export_policy_rule_resource.go pattern (index-based import):

Schema attributes:
- `id`: Computed, UseStateForUnknown (server-assigned UUID)
- `policy_name`: Required, RequiresReplace
- `name`: Computed, UseStateForUnknown (server-assigned rule name)
- `index`: Computed — server-assigned ordering index, used for import
- `client`: Required — IP/CIDR or "*"
- `effect`: Required — "allow" or "deny", stringvalidator.OneOf. Mutable (unlike OAP — NAP rule effect CAN be patched per research)
- `interfaces`: Optional, Computed, List of strings — e.g. ["nfs", "smb", "s3"]
- `policy_version`: Computed
- `timeouts`: standard

CRUD methods:
- Create: POST with policyName as query param. Body has client, effect, interfaces. Read back by created.Name.
- Read: GET by policyName+ruleName. Drift detection on client, effect.
- Update: PATCH by policyName+ruleName. Build patch comparing plan vs state for client, effect, interfaces.
- Delete: DELETE by policyName+ruleName.
- ImportState: Parse "policy_name/index" (SplitN "/" 2). Convert index to int. Call GetNetworkAccessPolicyRuleByIndex. Same pattern as NFS rule import.

Helpers:
- `readIntoState(ctx, policyName, ruleName, *model) diag.Diagnostics`
- `mapNAPRuleToModel(ctx, rule, *model) diag.Diagnostics` — handle interfaces list
