Replicate the NFS rule resource pattern with CRITICAL differences:

Schema:
- `id` ‚Äî Computed, UseStateForUnknown
- `policy_name` ‚Äî Required, RequiresReplace
- `name` ‚Äî Computed, UseStateForUnknown (server-assigned ‚Äî this IS the stable identifier, unlike NFS where name is unstable)
- NO `index` attribute ‚Äî SMB rules have no ordering concept
- `principal` ‚Äî Required (user or group, e.g., "Everyone", "DOMAIN\\user")
- `change` ‚Äî Optional, Computed (default varies ‚Äî "allow" or "deny")
- `full_control` ‚Äî Optional, Computed
- `read` ‚Äî Optional, Computed
- `timeouts` ‚Äî standard CRUD

ImportState ‚Äî DIFFERENT from NFS:
1. Parse `strings.SplitN(req.ID, "/", 2)` ‚Äî format: `policy_name/rule_name`
2. parts[1] is the **rule name** (server-assigned string), NOT a numeric index
3. Call `GetSmbSharePolicyRuleByName(ctx, policyName, ruleName)`
4. Set policy_name and name in state, read full state

Create/Read/Update/Delete: same pattern as NFS rule but using SMB client methods and SMB model types.
