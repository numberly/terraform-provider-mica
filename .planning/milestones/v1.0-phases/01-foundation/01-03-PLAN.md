**internal/client/filesystems.go:**

All methods on `*FlashBladeClient`. API paths use the client's baseURL (already includes `/api/2.22`).

1. `GetFileSystem(ctx, name string) (*FileSystem, error)`:
   - GET `/file-systems?names={name}`
   - Parse `ListResponse[FileSystem]`, return first item or IsNotFound error if empty.

2. `ListFileSystems(ctx, opts ListFileSystemsOpts) ([]FileSystem, error)`:
   - GET `/file-systems` with optional query params: `names`, `filter`, `destroyed` (bool), `continuation_token`, `limit`.
   - `ListFileSystemsOpts` struct with optional fields.
   - Return items slice.

3. `PostFileSystem(ctx, body FileSystemPost) (*FileSystem, error)`:
   - POST `/file-systems` with JSON body.
   - Parse single-item response, return FileSystem.

4. `PatchFileSystem(ctx, id string, body FileSystemPatch) (*FileSystem, error)`:
   - PATCH `/file-systems?ids={id}` with JSON body.
   - Use `id` (not name) for PATCH — stable across renames per CONTEXT.md locked decision.
   - Only marshal non-nil pointer fields (use `json:"...,omitempty"` — but note Go `omitempty` on `*bool` correctly omits nil but sends `false`).
   - Return updated FileSystem.

5. `DeleteFileSystem(ctx, id string) error`:
   - DELETE `/file-systems?ids={id}`.
   - Return nil on 200, error otherwise.

6. `PollUntilEradicated(ctx, name string) error`:
   - Loop: GET `/file-systems?names={name}&destroyed=true`.
   - If 404 or empty items → return nil (eradicated).
   - If still present → sleep 2s, retry.
   - Respect context deadline/cancellation.
   - Max poll duration: use context timeout (caller provides via Terraform resource timeout).

**Tests** use `httptest.NewServer` with hand-written handlers. Each test creates a minimal server for the specific endpoint being tested.

**internal/testmock/server.go:**

Create `NewMockServer() *MockServer` factory that:
1. Creates an `httptest.NewServer` with a `http.ServeMux`.
2. Registers common endpoints:
   - `POST /login` — returns `x-auth-token: mock-session-token` header.
   - `GET /api/api_version` — returns `{"versions": ["2.22"]}`.
3. Returns `MockServer` struct with fields: `Server *httptest.Server`, `Mux *http.ServeMux`, `URL() string`.
4. `Close()` method wraps `Server.Close()`.
5. Method `RegisterHandler(pattern, handler)` for adding resource-specific handlers.

**internal/testmock/handlers/filesystems.go:**

`RegisterFileSystemHandlers(mux *http.ServeMux)` that adds:

1. In-memory state: `map[string]*FileSystem` (keyed by name), `map[string]*FileSystem` (keyed by ID for PATCH/DELETE).
2. `GET /api/2.22/file-systems` — supports `?names=`, `?ids=`, `?destroyed=` query params. Returns `ListResponse[FileSystem]`. Returns 200 with empty items if not found (FlashBlade API behavior — client interprets empty as not found).
3. `POST /api/2.22/file-systems` — creates file system with generated UUID ID, sets computed fields (created timestamp, space with zero values, default NFS/SMB config). Returns 200 with created FileSystem.
4. `PATCH /api/2.22/file-systems` — requires `?ids=` param. Merges non-null fields. Supports rename (name field). Supports `destroyed` flag. Returns updated FileSystem.
5. `DELETE /api/2.22/file-systems` — requires `?ids=` param. Only works on destroyed file systems (returns 400 if not destroyed). Removes from state entirely (simulates eradication).

Thread-safe: use `sync.Mutex` on the in-memory state.

This mock server will be reused in Plan 04 for provider-level resource tests and in Phase 2+ for additional resource families.
