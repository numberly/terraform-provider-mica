**internal/client/errors.go:**
- Define `APIError` struct: StatusCode int, Message string, Errors []APISubError.
- Implement `error` interface on APIError.
- `IsNotFound(err) bool` — checks StatusCode == 404.
- `IsRetryable(statusCode int) bool` — true for 429, 503, and 5xx >= 500.
- `parseAPIError(resp *http.Response) error` — reads body, parses JSON error response from FlashBlade API format `{"errors": [{"message": "..."}]}`.

**internal/client/models.go:**
- Plain Go structs (no Terraform types) for FlashBlade API JSON:
  - `FileSystem` — all fields from GET response including `name`, `id`, `provisioned`, `destroyed`, `time_remaining`, `created`, `space` (nested), `nfs`, `smb`, `http`, `default_quotas`, `multi_protocol`, `promotion_status`, `request_number`, `source`, `writable`.
  - `Space` — `data_reduction`, `snapshots`, `total_physical`, `unique`, `virtual`, `snapshots_effective`.
  - `NFSConfig` — `enabled`, `v3_enabled`, `v4_1_enabled`, `rules`, `transport`.
  - `SMBConfig` — `enabled`, `access_based_enumeration_enabled`, `continuous_availability_enabled`, `smb_encryption_enabled`.
  - `FileSystemPost` — name, provisioned, nfs, smb, and writable fields only.
  - `FileSystemPatch` — all mutable fields (pointers for omit-empty semantics).
  - `ListResponse[T any]` — generic wrapper for `{"items": [...], "continuation_token": "...", "total_item_count": N}`.
  - `VersionResponse` — `{"versions": [string]}`.
  - JSON tags on all fields matching FlashBlade API snake_case names.
  - Use `*bool` for optional boolean fields in Patch structs so `false` is distinguishable from absent.

**internal/client/transport.go:**
- `retryTransport` struct implementing `http.RoundTripper`: wraps base transport, retries on retryable status codes.
- Fields: `base http.RoundTripper`, `maxRetries int`, `baseDelay time.Duration`.
- `RoundTrip` method: call base, check status, if `IsRetryable(status)` and retries remain, sleep with exponential backoff (baseDelay * 2^attempt, capped at 30s), retry. Drain and close response body before retry.
- Add `X-Request-ID` header (UUID v4 via `crypto/rand`) to every request for traceability.
- Log each retry attempt at Debug level using stdlib `log` (client layer has no tflog — that is provider-layer concern).

**internal/client/auth.go:**
- `LoginWithAPIToken(ctx, httpClient, endpoint, apiToken) (sessionToken string, err error)`:
  - POST to `{endpoint}/login` with header `api-token: {apiToken}`.
  - Extract `x-auth-token` from response header.
  - Return session token string.
- `FlashBladeTokenSource` struct implementing `oauth2.TokenSource`:
  - Fields: endpoint, apiToken, httpClient, mu sync.Mutex, cachedToken *oauth2.Token.
  - `Token()` — if cachedToken valid, return it. Otherwise POST to `{endpoint}/oauth2/1.0/token` with form data: `grant_type=urn:ietf:params:oauth:grant-type:token-exchange`, `subject_token={apiToken}`, `subject_token_type=urn:ietf:params:oauth:token-type:jwt`.
  - Parse `{"access_token": "...", "expires_in": N, "token_type": "Bearer"}` response.
  - Cache token with proper expiry.

**internal/client/client.go:**
- `const APIVersion = "2.22"`
- `Config` struct: Endpoint, APIToken, OAuth2ClientID, OAuth2KeyID, OAuth2Issuer, MaxRetries (default 3), RetryBaseDelay (default 1s), CACertFile, CACert, InsecureSkipVerify.
- `FlashBladeClient` struct: httpClient *http.Client, baseURL string (endpoint + "/api/" + APIVersion), sessionToken string.
- `NewClient(cfg Config) (*FlashBladeClient, error)`:
  1. Validate endpoint not empty.
  2. Call `buildTransport(cfg)` to get TLS+retry RoundTripper.
  3. If APIToken set: call `LoginWithAPIToken` to get session token, set as default header.
  4. If OAuth2 fields set: create `FlashBladeTokenSource`, wrap httpClient with `oauth2.NewClient`.
  5. Return `FlashBladeClient`.
- `buildTransport(cfg Config) (http.RoundTripper, error)` — build TLS config with custom CA from file/inline, wrap in retryTransport. Per RESEARCH.md Pattern 7.
- `NegotiateVersion(ctx) error` — GET `/api/api_version`, verify `"2.22"` in versions array. Per RESEARCH.md Pattern 5.
- Helper methods: `get(ctx, path, result)`, `post(ctx, path, body, result)`, `patch(ctx, path, body, result)`, `delete(ctx, path)` — all add `X-Auth-Token` or `Authorization: Bearer` header, set Content-Type application/json, handle response parsing and error extraction.

**Tests:** Use `httptest.NewServer` for all tests. No external dependencies. Create test CA cert using `crypto/x509` test helpers for TLS tests.
