  <name>Task 1: Add client model structs for eradication, object lock, and public access configs</name>
  <files>internal/client/models_storage.go</files>
  <action>
Add 3 new structs to models_storage.go:

1. **EradicationConfig** — JSON struct with fields:
   - `EradicationDelay` int64 `json:"eradication_delay,omitempty"` (milliseconds)
   - `EradicationMode` string `json:"eradication_mode,omitempty"` (e.g., "retention-based", "permission-based")
   - `ManualEradication` string `json:"manual_eradication,omitempty"` (e.g., "enabled", "disabled")

2. **ObjectLockConfig** — JSON struct with fields:
   - `FreezeLockgedObjects` bool `json:"freeze_locked_objects,omitempty"`
   - `DefaultRetention` int64 `json:"default_retention,omitempty"` (seconds)
   - `DefaultRetentionMode` string `json:"default_retention_mode,omitempty"` (e.g., "compliance", "governance")
   - `ObjectLockEnabled` bool `json:"object_lock_enabled,omitempty"`
   NOTE: The bool fields need special handling for PATCH — see BucketPatch below.

3. **PublicAccessConfig** — JSON struct with fields:
   - `BlockNewPublicPolicies` bool `json:"block_new_public_policies,omitempty"`
   - `BlockPublicAccess` bool `json:"block_public_access,omitempty"`
   NOTE: Same bool PATCH concern — see below.

Update existing structs:

4. **Bucket** (GET response) — add fields:
   - `EradicationConfig EradicationConfig json:"eradication_config"`
   - `ObjectLockConfig ObjectLockConfig json:"object_lock_config"`
   - `PublicAccessConfig PublicAccessConfig json:"public_access_config"`
   - `PublicStatus string json:"public_status,omitempty"`
   No omitempty on config objects — API always returns them on GET.

5. **BucketPost** — add fields:
   - `EradicationConfig *EradicationConfig json:"eradication_config,omitempty"`
   - `ObjectLockConfig *ObjectLockConfig json:"object_lock_config,omitempty"`
   (public_access_config is NOT valid on POST per API spec — PATCH only)

6. **BucketPatch** — add pointer fields for PATCH semantics:
   - `EradicationConfig *EradicationConfig json:"eradication_config,omitempty"`
   - `ObjectLockConfig *ObjectLockConfig json:"object_lock_config,omitempty"`
   - `PublicAccessConfig *PublicAccessConfig json:"public_access_config,omitempty"`
   Using pointer-to-struct allows omitting the entire block when unchanged. The inner bool fields use omitempty which is fine because the API treats the entire nested object as a unit on PATCH (send the full config object or omit it entirely).

**A. bucketModel struct** — add 4 new fields:
```go
EradicationConfig  types.Object `tfsdk:"eradication_config"`

PublicStatus       types.String `tfsdk:"public_status"`
```

**B. Schema()** — add to the Attributes map:

1. `"eradication_config"`: `schema.SingleNestedAttribute` with Optional+Computed, containing:
   - `eradication_delay` (Int64, Optional+Computed) — eradication delay in milliseconds
   - `eradication_mode` (String, Optional+Computed) — "retention-based" or "permission-based"
   - `manual_eradication` (String, Optional+Computed) — "enabled" or "disabled"

2. `"object_lock_config"`: `schema.SingleNestedAttribute` with Optional+Computed, containing:
   - `freeze_locked_objects` (Bool, Optional+Computed)
   - `default_retention` (Int64, Optional+Computed) — default retention in seconds
   - `default_retention_mode` (String, Optional+Computed) — "compliance" or "governance"
   - `object_lock_enabled` (Bool, Optional+Computed)

3. `"public_access_config"`: `schema.SingleNestedAttribute` with Optional+Computed, containing:
   - `block_new_public_policies` (Bool, Optional+Computed)
   - `block_public_access` (Bool, Optional+Computed)

4. `"public_status"`: `schema.StringAttribute` with Computed only, Description "Bucket's public access status" and `stringplanmodifier.UseStateForUnknown()`.
