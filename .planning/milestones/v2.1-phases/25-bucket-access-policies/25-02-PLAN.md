**flashblade_bucket_access_policy (bucket_access_policy_resource.go):**

Resource struct: `bucketAccessPolicyResource` with `client *client.FlashBladeClient`.
Interfaces: Resource, ResourceWithConfigure, ResourceWithImportState.
No ResourceWithUpgradeState needed (version 0, new resource).

Model struct `bucketAccessPolicyModel`:
- ID types.String (computed, UseStateForUnknown)
- BucketName types.String (required, RequiresReplace — policy is per-bucket, cannot move)
- Enabled types.Bool (computed — API manages this)
- Timeouts timeouts.Value

Schema: Type name "flashblade_bucket_access_policy". Description: "Manages a FlashBlade bucket access policy for IAM-style per-bucket authorization."

CRUD:
- Create: Read plan. Build BucketAccessPolicyPost (empty rules — rules managed via separate resource). Call client.PostBucketAccessPolicy(ctx, bucketName, body). Map response to model.
- Read: Get state. Call client.GetBucketAccessPolicy(ctx, bucketName). If not found, RemoveResource. Map response to model.
- Update: NOT needed — no PATCH endpoint. Do NOT implement ResourceWithModifyPlan. The only mutable-looking field (enabled) is API-managed. If user changes bucket_name, RequiresReplace triggers destroy+create.
- Delete: Get state. Call client.DeleteBucketAccessPolicy(ctx, bucketName). Handle not-found gracefully (already deleted).
- ImportState: Parse single-part ID as bucket_name. Call GetBucketAccessPolicy. Map to model with nullTimeoutsValue().

Mapper: `mapBucketAccessPolicyToModel(policy *client.BucketAccessPolicy, data *bucketAccessPolicyModel)` — set ID, BucketName from policy.Bucket.Name, Enabled.

**flashblade_bucket_access_policy_rule (bucket_access_policy_rule_resource.go):**

Resource struct: `bucketAccessPolicyRuleResource` with `client *client.FlashBladeClient`.
Interfaces: Resource, ResourceWithConfigure, ResourceWithImportState.

Model struct `bucketAccessPolicyRuleModel`:
- Name types.String (computed, UseStateForUnknown — auto-generated by API)
- BucketName types.String (required, RequiresReplace)
- Actions types.List of types.StringType (required)
- Effect types.String (computed — always "allow", read-only)
- Principals types.List of types.StringType (required) — flattened from principals.all array
- Resources types.List of types.StringType (required)
- Timeouts timeouts.Value

Schema: Type name "flashblade_bucket_access_policy_rule". Description: "Manages an individual rule within a FlashBlade bucket access policy."

CRUD:
- Create: Read plan. Convert types.List fields to Go slices. Build BucketAccessPolicyRulePost with Actions, Effect="allow", Principals (wrap in BucketAccessPolicyPrincipals{All: principals}), Resources. Call client.PostBucketAccessPolicyRule(ctx, bucketName, body). Map response to model.
- Read: Get state. Call client.GetBucketAccessPolicyRule(ctx, bucketName, name). If not found, RemoveResource. Map response to model.
- Update: NOT needed — no PATCH endpoint. All fields are RequiresReplace or computed, so changes trigger destroy+create.
- Delete: Get state. Call client.DeleteBucketAccessPolicyRule(ctx, bucketName, name). Handle not-found gracefully.
- ImportState: Parse composite ID "bucketName/ruleName" via parseCompositeID(id, 2). Call GetBucketAccessPolicyRule. Map to model with nullTimeoutsValue().

Mapper: `mapBucketAccessPolicyRuleToModel(rule *client.BucketAccessPolicyRule, data *bucketAccessPolicyRuleModel)` — set Name, BucketName from rule.Policy.Name (or extract from context), Actions as types.ListValueFrom, Effect, Principals from rule.Principals.All as types.ListValueFrom, Resources as types.ListValueFrom.

IMPORTANT: For types.List conversion, use `types.ListValueFrom(ctx, types.StringType, slice)` pattern from the framework. Handle diagnostics properly.
