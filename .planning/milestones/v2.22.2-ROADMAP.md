## Overview

Added two resources and one data source for FlashBlade LDAP role-based access control through Terraform:

- `flashblade_directory_service_role` — maps an LDAP group to a built-in FlashBlade management role (`array_admin`, `storage_admin`, `ops_admin`, `readonly`). Full CRUD, import by name, drift detection, data source. User-supplied name (fixed in 50.1 after initial "server-generated" design was invalidated by real-array testing — the API actually requires `?names=`).
- `flashblade_management_access_policy_directory_service_role_membership` — additive policy↔role association. Composite ID `<role_name>/<policy_name>` (slash separator required because role names can contain `:`, e.g. `pure:policy/array_admin`). Destroy severs the link only; neither the policy nor the role is deleted.

Phase 50.1 was inserted after the initial Phase 50 ship because live-array UAT on par5/pa7 revealed the POST endpoint actually requires `?names=<name>` (contrary to the swagger doc and initial D-03 design). Fix required client signature change, mock handler update, schema v0→v1 upgrader, and docs regeneration.

## Milestone Summary

**Key Decisions:**
- **D-03 (superseded in 50.1):** Initial design modeled DSR name as server-generated (per swagger doc). Reverted after live-array UAT proved `?names=` is required.
- **D-05:** Composite ID for DSRM membership uses `/` (not `:`) separator because role names can contain `:` (e.g. `pure:policy/array_admin`). Import path: `<role_name>/<policy_name>`.
- Resource surface: DSR is a full resource (CRUD); DSRM is additive-only (no Update — changes to `role` or `policy` trigger `RequiresReplace`).
- Drift policy: `management_access_policies` field on DSR is a computed list; drift is logged via `tflog.Debug` with `{resource, field, was, now}`.

**Issues Resolved:**
- Live-array `HTTP 400 Names query parameter is required` on POST (root cause: swagger doc misrepresents endpoint contract) — fixed in 50.1-01.
- DSRM membership endpoint: swagger/initial impl used `role_names` query param, real API expects `member_names` — fixed in `05faac1`.
- Certificate validation regression (unrelated but surfaced during UAT) — fixed in `d67484c`.

**Issues Deferred (Known Limitations / Backlog):**
- No `VALIDATION.md` (Nyquist formal coverage) for phase 50 or 50.1. Same pattern as v2.22.1 — tracked as backlog, non-blocking. `workflow.nyquist_validation: true` but no retrofit planned.
- Role scope/locality attributes not exposed (API does not yet surface multi-array role propagation).
- Bulk role import (HCL `for_each` from LDAP group list) — usability helper, not a provider requirement.
- `/directory-services/test` ephemeral resource (dry-run LDAP bind) — deferred to its own milestone.

**Technical Debt Incurred:**
- None beyond the Nyquist backlog item above. All 3 doc-drift items surfaced by audit (models_admin.go stale comment, ROADMAP Notes column, CONVENTIONS baseline) were fixed pre-archival in commit `862462e`.
