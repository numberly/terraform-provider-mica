<objective>
Harden error classification to handle wrapped errors, fix ParseAPIError edge case, add fresh-GET guard to bucket delete, and replace JSON round-trip in test helper with reflect.

Purpose: Errors wrapped with fmt.Errorf("%w") currently bypass IsNotFound/IsConflict/IsUnprocessable because they use direct type assertion instead of errors.As. ParseAPIError silently swallows io.ReadAll failures. Bucket delete relies on potentially stale ObjectCount from state. Test helper countItems uses an unnecessary JSON marshal/unmarshal round-trip.

Output: Resilient error classification across all error paths, safer bucket deletion, cleaner test helpers.
</objective>
