## CONVENTIONS.md Evolution

Three extensions during the milestone:

1. **POST struct exceptions** (pre-phase-51): `*bool` allowed when API default is non-zero; `*int64`/`*string` for semantic zero; pointers on nested structs and optional refs are the correct Go idiom for `omitempty`.
2. **Resource Implementation exception** (phase 52-03): access key reject-shim pattern documented with rationale.
3. **PATCH slice `*[]T`** (phase 53-01) with `omitempty`, plus **"always send" exception** carve-out for endpoints where absent ≠ empty list (applied to `NetworkInterfacePatch`).

## Key Decisions

- **D-51-01** `doublePointerRefForPatch(state, plan types.String) **client.NamedReference` helper — single source of truth for all `**NamedReference` PATCH semantics going forward.
- **D-51-04** State upgraders on schema version bumps are no-op identity transforms when only the wire format changes (Terraform attribute types unchanged). Defensive by convention, not by framework requirement.
- **D-51-06** `R-001 + R-002` (two changes on SubnetPost/Patch) merged into a single subnet `v0→v1` bump to avoid a `v0→v1→v2` chain for one resource.
- **D-52-03** Access key is the only documented exception to the 4-interface rule; `ImportState` rejects at runtime with an informative error.
- **D-53-02** `NetworkInterfacePatch` stays with plain slices + no `omitempty`; CONVENTIONS.md formalizes the carve-out rather than migrating and breaking working code.
