## Decisions Made

- **SingleModule**: KnownModules with named modules breaks on `flashblade_bucket` (name="" after stripping module prefix). SingleModule produces `flashblade:index:*` tokens for all resources.
- **schema-embed.json non-existence**: Bridge v3.127.0 tfgen only produces `schema.json` + `bridge-metadata.json`. The `schema-embed.json` concept from some docs is a v3 era artifact — current pattern is to embed `schema.json` directly.
- **Sensitive ID**: `Secret=False()` acknowledges the ID will be exposed in state (IDs can't be encrypted). The bridge error message "set Secret=True()" is wrong — the test suite confirms False() is the fix.
- **Nested config**: TF provider has `auth.api_token`, `auth.oauth2.*` (nested). Top-level Config overrides for these names were producing "not present in Terraform metadata" warnings. Removed — bridge auto-promotes nested Sensitive=true fields.
