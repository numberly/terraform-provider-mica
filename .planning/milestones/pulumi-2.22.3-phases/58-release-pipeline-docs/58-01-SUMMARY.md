# Phase 58 Plan 01: Pulumi Release Pipeline Summary

One-liner: GoReleaser config for cosign-signed multi-platform Pulumi plugin binaries plus GitHub Actions workflow triggering on `pulumi-*` tags, building provider, attaching Python wheel, and pushing Go SDK submodule tag.

## What Was Built

### Task 1: Pulumi GoReleaser Config

Created `pulumi/.goreleaser.pulumi.yml` with:
- `project_name: pulumi-resource-flashblade`
- 5 platform builds: linux/darwin/windows x amd64 + linux/darwin x arm64
- `tar.gz` archives named `pulumi-resource-flashblade-v{VERSION}-{os}-{arch}.tar.gz`
- Embedded `schema.json` + `bridge-metadata.json` via `archives.files`
- Keyless Sigstore cosign signing (`cosign sign-blob --yes`)
- Release header with install instructions for plugin, Python wheel, and Go SDK
- Changelog grouped by conventional commit types

### Task 2: Pulumi Release Workflow

Created `.github/workflows/pulumi-release.yml` with:
- Trigger on `pulumi-*` tags
- 4-job dependency chain:
  1. **prerequisites** — unit tests, `make tfgen`, schema drift gate, upload schema + bridge-metadata artifacts
  2. **release_provider** (needs prerequisites) — download artifacts, install cosign, run GoReleaser
  3. **release_python_sdk** (needs prerequisites) — generate Python SDK, build wheel, upload to release
  4. **tag_go_sdk** (needs release_provider) — push `sdk/go/vX.Y.Z` git tag
- Permissions: `contents: write`, `id-token: write`, `attestations: write`
