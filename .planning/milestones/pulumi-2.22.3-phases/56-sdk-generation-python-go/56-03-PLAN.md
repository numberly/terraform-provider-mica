# Phase 56, Plan 03: Go SDK Generation and Compilation

## Objective

Generate a compilable Go SDK under `pulumi/sdk/go/` with module path `github.com/numberly/opentofu-provider-flashblade/pulumi/sdk/go`. Verify `go build ./...` exits 0.

## Context

The Go SDK directory `pulumi/sdk/go/` exists with a `go.mod` but no generated code. The module path is `github.com/numberly/opentofu-provider-flashblade/pulumi/sdk/go` and it requires `github.com/pulumi/pulumi/sdk/v3 v3.231.0`.

Go SDK generation for Pulumi bridged providers typically produces:
- `flashblade/init.go` — package init with resource registrations
- `flashblade/*.go` — one file per resource and data source
- `flashblade/pulumiTypes.go` — type definitions
- `flashblade/pulumiUtilities.go` — utility functions
- `flashblade/config/config.go` — provider config types

The generated code must compile with the existing `go.mod`.

</read_first>
<action>
Build the Go SDK to verify it compiles:

```bash
cd pulumi/sdk/go
go build ./...
```

If compilation fails:
1. Read the compiler errors
2. Determine if errors are in generated code (regenerate) or missing dependencies (add deps)
3. Fix and retry until `go build ./...` exits 0

Common issues:
- Missing `go.sum` entries → run `go mod tidy`
- Type mismatches in generated code → may indicate schema incompatibilities; log and fix
- Import path mismatches → ensure module path in go.mod matches generation output
</action>
<acceptance_criteria>

- No compiler errors
- All generated `.go` files compile successfully
</acceptance_criteria>
</task>
