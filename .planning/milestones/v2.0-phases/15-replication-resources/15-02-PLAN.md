**remoteCredentialsStore** struct:
- `mu sync.Mutex`
- `byName map[string]*client.ObjectStoreRemoteCredentials`
- `nextID int` (for generating unique IDs like "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx" format, or simpler: "rc-1", "rc-2")
- Reference to `arrayConnectionStore` for validating remote_names (optional, can skip validation in mock)

**RegisterRemoteCredentialsHandlers(mux *http.ServeMux) *remoteCredentialsStore** — register at `/api/2.22/object-store-remote-credentials`.

**handleGet**: Filter by `?names=` query param. Return items with `secret_access_key` stripped (empty string) — same pattern as access keys. Validate query params using `ValidateQueryParams` helper.

**handlePost**: Read `?names=` and `?remote_names=` from query params. Decode body for access_key_id + secret_access_key. Generate an ID. Store credential with Remote.Name set from remote_names param. Return the full credential including secret_access_key (POST only, like access keys).

**handlePatch**: Read `?names=` from query params. Decode body. Update access_key_id and/or secret_access_key if present. Return updated credential (without secret in response, since PATCH response behaves like GET).

**handleDelete**: Read `?names=` from query params. Delete from store. Return 200.

Validate mandatory query params: GET requires nothing mandatory, POST requires `names` + `remote_names`, PATCH requires `names`, DELETE requires `names`.

**Create internal/provider/remote_credentials_resource.go** following the object_store_account_export_resource.go pattern:

Terraform model struct `remoteCredentialsModel`:
- `ID` types.String (Computed, UseStateForUnknown)
- `Name` types.String (Required, RequiresReplace — identifies the credential)
- `AccessKeyID` types.String (Required, Sensitive)
- `SecretAccessKey` types.String (Required, Sensitive)
- `RemoteName` types.String (Required, RequiresReplace — reference to array connection remote name)
- `Timeouts` timeouts.Value

Resource struct: `remoteCredentialsResource` with `client *client.FlashBladeClient`.
Type name: `flashblade_object_store_remote_credentials`.
Implement: Resource, ResourceWithConfigure, ResourceWithImportState, ResourceWithUpgradeState.
SchemaVersion: 0, UpgradeState returns empty map.

**Create**: Call `r.client.PostRemoteCredentials(ctx, name, remoteName, post)`. Map response to model. Note: secret_access_key is in the POST response but NOT in GET. Since the user provides it in HCL, we keep the plan value in state (not the API response) for access_key_id and secret_access_key — they are user-managed, not API-managed.

**Read**: Call `r.client.GetRemoteCredentials(ctx, name)`. Map response to model. IMPORTANT: access_key_id comes from API response. secret_access_key is NOT returned by GET — preserve state value (do not overwrite with empty string). Remote name comes from response.Remote.Name.

**Update**: Build ObjectStoreRemoteCredentialsPatch from changed fields (access_key_id, secret_access_key). Call PatchRemoteCredentials. Then re-read to refresh computed fields. Preserve secret_access_key in state from plan value (not API response).

**Delete**: Call `r.client.DeleteRemoteCredentials(ctx, name)`. Handle IsNotFound gracefully.

**ImportState**: Import by name. Call GetRemoteCredentials. Map to model. Note: secret_access_key will be empty after import — user must provide it in config or use `ignore_changes`. Set Timeouts to null object.

**Mapping function** `mapRemoteCredentialsToModel(cred *client.ObjectStoreRemoteCredentials, data *remoteCredentialsModel)`:
- data.ID = cred.ID
- data.Name = cred.Name
- data.AccessKeyID = cred.AccessKeyID
- data.RemoteName = cred.Remote.Name
- Do NOT set data.SecretAccessKey from API response (it's empty on GET)

- No secret_access_key in data source (sensitive, not returned by API anyway)
- Read: GetRemoteCredentials by name, map to model
- Type name: `flashblade_object_store_remote_credentials`
