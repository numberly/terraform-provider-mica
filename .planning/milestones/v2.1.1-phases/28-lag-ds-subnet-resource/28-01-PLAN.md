Create `internal/client/models_network.go` with these structs (copy exact field definitions from 28-RESEARCH.md "Model Struct Design" section):

1. `Subnet` -- full GET response struct with all fields (ID, Name, Enabled, Gateway, Interfaces []NamedReference, LinkAggregationGroup *NamedReference, MTU, Prefix, Services []string, VLAN)
2. `SubnetPost` -- writable fields only: Gateway(string), LinkAggregationGroup(*NamedReference), MTU(int64), Prefix(string), VLAN(int64). NO Name, NO Enabled, NO Interfaces, NO Services, NO ID.
3. `SubnetPatch` -- pointer types for partial update: Gateway(*string), LinkAggregationGroup(*NamedReference), MTU(*int64), Prefix(*string), VLAN(*int64). Use `*int64` for MTU and VLAN to handle zero-value correctly (vlan=0 is untagged, must be serializable).
4. `LinkAggregationGroup` -- read-only GET struct: ID, Name, LagSpeed(int64), MacAddress(string), PortSpeed(int64), Ports([]string), Status(string). All fields `json:"...,omitempty"` except Name.

Create `internal/client/subnets.go` with 5 methods following the servers.go pattern:
- `GetSubnet(ctx, name)` -- uses `getOneByName[Subnet]` with path `/subnets?names=`
- `ListSubnets(ctx)` -- GET `/subnets`, returns `[]Subnet`
- `PostSubnet(ctx, name, body SubnetPost)` -- POST `/subnets?names=` (NOT `?create_ds=`, subnets use `?names=` per research)
- `PatchSubnet(ctx, name, body SubnetPatch)` -- PATCH `/subnets?names=`
- `DeleteSubnet(ctx, name)` -- DELETE `/subnets?names=`, returns error only (no cascade)

Create `internal/client/link_aggregation_groups.go` with 2 GET-only methods:
- `GetLinkAggregationGroup(ctx, name)` -- uses `getOneByName[LinkAggregationGroup]` with path `/link-aggregation-groups?names=`
- `ListLinkAggregationGroups(ctx)` -- GET `/link-aggregation-groups`, returns `[]LinkAggregationGroup`

NO POST/PATCH/DELETE for LAGs -- they are hardware-managed.

Create `internal/testmock/handlers/subnets.go` following the servers.go mock pattern:

1. `subnetStore` struct with `sync.Mutex`, `byName map[string]*client.Subnet`, `byID map[string]*client.Subnet`
2. `RegisterSubnetHandlers(mux *http.ServeMux) *subnetStore` -- registers handler at `/api/2.22/subnets`
3. `AddSubnet(name string, prefix string, lagName string) *client.Subnet` -- seeder method for tests. Sets reasonable defaults: Enabled=true, MTU=1500, VLAN=0, generates UUID for ID. If lagName != "", sets LinkAggregationGroup to NamedReference.
4. `handle(w, r)` -- dispatches by HTTP method (GET/POST/PATCH/DELETE)
5. `handleGet` -- filters by `?names=` query param. If names present, return matching subnet or empty list. If no names, return all subnets. Uses `WriteJSONListResponse`.
6. `handlePost` -- reads `?names=` query param (NOT `?create_ds=`). Decodes body as `client.SubnetPost`. Creates `client.Subnet` with user-provided name. Checks for conflict (409). Sets defaults: Enabled=true, generated UUID. Stores in both maps. Returns created subnet via `WriteJSONListResponse`.
7. `handlePatch` -- reads `?names=` query param. Uses `map[string]json.RawMessage` raw decode for true PATCH semantics. Updates only provided fields on the existing subnet. For `link_aggregation_group`, unmarshal to `*client.NamedReference`. For `mtu`/`vlan`, unmarshal to `int64`. For `gateway`/`prefix`, unmarshal to `string`.
8. `handleDelete` -- reads `?names=` query param. Removes from both maps. Returns 200 on success, 404 if not found.

Create `internal/testmock/handlers/link_aggregation_groups.go` following the read-only pattern:

1. `lagStore` struct with `sync.Mutex`, `lags map[string]*client.LinkAggregationGroup`
2. `RegisterLinkAggregationGroupHandlers(mux *http.ServeMux) *lagStore` -- registers handler at `/api/2.22/link-aggregation-groups`, GET only
3. `Seed(lag *client.LinkAggregationGroup)` -- inserts a LAG by name for test setup
4. `handleGet` -- filters by `?names=` query param. Returns matching LAG or empty list. Uses `WriteJSONListResponse`.
5. No POST/PATCH/DELETE handlers -- return 405 Method Not Allowed for non-GET.
