    Create a full-stack networking workflow example in `examples/workflows/networking/` following the established pattern from `examples/workflows/s3-tenant-full-stack/`.

    **main.tf structure:**

    1. Header comment block with ASCII architecture diagram:
       ```
       # Architecture:
       #
       #   [LAG] (data source — hardware-managed)
       #       |
       #   [Subnet] (layer-3 network segment on the LAG)
       #       |
       #       +-- [VIP: data] --> attached to server (client data traffic)
       #       +-- [VIP: sts] --> attached to server (S3 STS traffic)
       #       +-- [VIP: egress-only] --> no server (outbound replication)
       #       |
       #   [Server] (data source — reads back attached VIPs)
       ```

    2. `terraform {}` block with required_providers (flashblade, source "numberly/flashblade", version "~> 2.1")

    3. Variables: flashblade_endpoint (string), flashblade_api_token (string, sensitive), lag_name (string, default "lag1"), subnet_name (string), subnet_prefix (string), subnet_gateway (string), server_name (string)

    4. Provider block with endpoint, auth, insecure_skip_verify

    5. Steps with section comments (matching s3-tenant-full-stack style):

       **Step 1: Read the LAG** — `data "flashblade_link_aggregation_group" "this"` by lag_name variable

       **Step 2: Create subnet** — `resource "flashblade_subnet" "this"` with name from var, prefix, gateway, mtu 9000, vlan 100, lag_name referencing the LAG data source name

       **Step 3: Create VIPs** — Three network interfaces:
       - `flashblade_network_interface.data_vip`: address "10.0.0.10", subnet_name from subnet resource, type "vip", services "data", attached_servers = [var.server_name]. Comment: "Client data traffic — must be attached to a server"
       - `flashblade_network_interface.sts_vip`: address "10.0.0.11", same subnet, type "vip", services "sts", attached_servers = [var.server_name]. Comment: "S3 STS traffic — must be attached to a server"
       - `flashblade_network_interface.egress_vip`: address "10.0.0.12", same subnet, type "vip", services "egress-only". Comment: "Outbound replication — no server attachment required"

       **Step 4: Read server with VIPs** — `data "flashblade_server" "this"` by server_name. Add `depends_on = [flashblade_network_interface.data_vip, flashblade_network_interface.sts_vip]` to ensure VIPs are created before reading.

    6. Outputs: lag_name, subnet_name, data_vip_address, sts_vip_address, egress_vip_address, server_network_interfaces (from server data source)

    **terraform.tfvars.example:**

    ```
    server_name          = "srv-my-server"
    ```
