  <files>internal/provider/validators.go, internal/provider/validators_test.go</files>
  <action>
Create `internal/provider/validators.go` with two custom `validator.String` implementations:

1. **alphanumericValidator** — rejects any string not matching `^[a-zA-Z0-9]+$`. Used for S3 export policy rule names. Error message: `"value must contain only alphanumeric characters (a-z, A-Z, 0-9)"`.

2. **hostnameNoDotValidator** — rejects any string containing a dot. Allowed: alphanumeric, hyphen, underscore. Regex: `^[a-zA-Z0-9_-]+$`. Used for virtual host hostnames. Error message: `"value must contain only alphanumeric characters, hyphens, and underscores (no dots)"`.

Both must implement the `github.com/hashicorp/terraform-plugin-framework/schema/validator` `String` interface (methods: `Description`, `MarkdownDescription`, `ValidateString`). Follow the pattern from the terraform-plugin-framework-validators library.

Export factory functions: `func AlphanumericValidator() validator.String` and `func HostnameNoDotValidator() validator.String`.

Create `internal/provider/validators_test.go` with table-driven tests:
- alphanumericValidator: accept "myname", "Rule01"; reject "my-name", "my.name", "my name", ""
- hostnameNoDotValidator: accept "my-host", "host_01", "host"; reject "my.host.com", "host.local", ""
