<objective>
Fix the resource schema (`name`: Computed → Required + RequiresReplace), bump SchemaVersion
0→1, add a v0→v1 state upgrader with the exact broken v0 PriorSchema so existing state files
can migrate, wire Create to pass the name to the updated PostDirectoryServiceRole, and update
resource tests to supply a name in HCL configs and add the StateUpgrade test.

Purpose: Without this fix the resource schema silently accepts no name from the user (Computed),
causing `terraform plan` to mark name as unknown and `terraform apply` to fail at POST. After
this fix, users must supply `name = "..."` in HCL — the resource is now fully correct.
Output: Fixed resource implementation, one new state upgrade test, updated lifecycle test.
