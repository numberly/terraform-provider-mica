<decisions>
## Implementation Decisions

### Client API (LOCKED)
- **D-01:** `PostDirectoryServiceRole` signature becomes:
  ```go
  func (c *FlashBladeClient) PostDirectoryServiceRole(ctx context.Context, name string, body DirectoryServiceRolePost) (*DirectoryServiceRole, error)
  ```
  Path: `"/directory-services/roles?names=" + url.QueryEscape(name)`. Matches the convention used by `PostTarget`, `PostRemoteCredentials`, etc.
- **D-02:** `DirectoryServiceRolePost` struct does NOT gain a `Name` field. Name is passed via query param (FlashBlade convention — `json:"-"` is used everywhere else; adding to struct would be inconsistent). If the struct currently has no `json:"-"` name field, leave it alone.
- **D-03:** Remove the misleading comment block at lines 15-18 ("API generates the role name server-side from the first management_access_policies entry"). Replace with standard one-line doc comment.

### Resource Schema (LOCKED)
- **D-04:** `name` attribute changes:
  - Before: `Computed: true, PlanModifiers: [UseStateForUnknown()]`
  - After: `Required: true, PlanModifiers: [stringplanmodifier.RequiresReplace()]`
  - Description update: "Unique name for the directory service role. Required on create. Changing this forces a new resource."
- **D-05:** `SchemaVersion` bumps 0→1. Add `UpgradeState` map with key `0` (prior version).
- **D-06:** V0 PriorSchema is an **exact copy** of the current (broken) v0 schema — `name` as Computed with UseStateForUnknown. Upgrader copies all fields including `name` as-is (existing state files will already have a name populated from the API response, so carry it forward verbatim).
- **D-07:** Intermediate model struct name: `directoryServiceRoleV0Model` (follows convention from `remote_credentials_resource.go` / `server_resource.go`).
- **D-08:** `Create` method adds name wiring: after `data.Timeouts` and before building `body`, pass `data.Name.ValueString()` to `PostDirectoryServiceRole`.
- **D-09:** `ImportState` — no functional change (already imports by `req.ID` which is the name). `data.Name = types.StringValue(name)` is already set via `mapDirectoryServiceRoleToModel`. Verify.

### Mock Handler (LOCKED)
- **D-10:** POST handler in `internal/testmock/handlers/directory_service_roles.go`:
  - Require `?names=<name>` query param via `RequireQueryParam(w, r, "names")` — 400 if missing
  - Parse the name from the query string and use it as the store key (instead of synthesizing from first policy)
  - Remove the synthesis logic that derives the name from `management_access_policies[0]` or a sequential fallback
  - Still enforce uniqueness: if name exists in `byName`, return 409 Conflict
- **D-11:** GET and PATCH/DELETE handlers unchanged — already use `?names=`.
