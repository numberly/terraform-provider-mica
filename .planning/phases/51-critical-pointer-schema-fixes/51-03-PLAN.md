<objective>
Fix R-003 on the file_system_export resource: `FileSystemExportPatch.Server` and `FileSystemExportPatch.SharePolicy` must be `**NamedReference`. Bump schema Version 0→1 with a no-op identity upgrader (R-005, D-51-03/04).

Purpose: Without `**NamedReference`, users cannot clear `share_policy_name` via `terraform apply` — the nil `*NamedReference` is indistinguishable from "don't touch" and `omitempty` drops it. This fix exposes the three-state CLEAR semantics to Terraform users through the new `doublePointerRefForPatch` helper.

Note: `server_name` carries `RequiresReplace` in the current schema — clearing it is not a Terraform-plan path in practice. We still convert it to `**NamedReference` to match the convention's §PATCH struct rule ("every field is a pointer") and keep the struct internally consistent. Update() continues to never send it (server_name changes force resource replacement).
