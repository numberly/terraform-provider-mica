<objective>
Fix the two critical pointer violations on the subnet resource:
- R-001: `SubnetPost.VLAN` must be `*int64` so VLAN=0 ("untagged") is sent explicitly instead of silently dropped by `omitempty`.
- R-002: `SubnetPatch.LinkAggregationGroup` must be `**NamedReference` so users can clear the LAG via `terraform apply`.
- R-005: Bump schema `Version` from 0 to 1 and add a no-op `v0→v1` state upgrader (per D-51-06, both changes go in a single version bump).

Purpose: Match CONVENTIONS.md §Pointer rules for POST (`*int64` for semantic-zero scalars) and PATCH (`**NamedReference` for clearable refs). Without these fixes, users cannot create untagged subnets with `vlan=0`, and cannot detach a subnet from a LAG.

Output:
- `models_network.go` with corrected `SubnetPost`/`SubnetPatch` struct fields
- `subnet_resource.go` with Version=1, UpgradeState, pointer-aware Create(), helper-based Update()
- `handlers/subnets.go` updated to decode POST VLAN as `*int64`
- 3 new tests (`StateUpgrade_V0toV1`, `Create_VLANZero`, `Patch_ClearLag`)
</objective>
