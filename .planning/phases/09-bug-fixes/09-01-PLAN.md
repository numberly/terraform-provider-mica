In the Delete method of object_store_account_export_resource.go (around line 262-264):

Current code:
```go
memberName := data.AccountName.ValueString()
exportName := data.Name.ValueString()
```

`data.Name` is the combined "account/export" format from the API. The `DeleteObjectStoreAccountExport` client method uses `exportName` in `names=` query param, which expects just the short export name.

Fix: Extract the short name from the combined name. The combined format is always "accountName/exportName", so split on "/" and take the last element:
```go
memberName := data.AccountName.ValueString()
combinedName := data.Name.ValueString()
// The API returns Name as "account/export" but Delete expects just the short export name.
exportName := combinedName
if idx := strings.LastIndex(combinedName, "/"); idx >= 0 {
    exportName = combinedName[idx+1:]
}
```

Add `"strings"` to imports if not already present.

Write a unit test in the existing test file that verifies the Delete lifecycle sends the correct short name. Use the existing mock server pattern from other resource tests in this codebase. The test should:
1. Create an account export via the mock
2. Verify the DELETE request receives `names=shortname` (not `names=account/shortname`)

    Computed:    true,
    Description: "Whether the file system is writable.",
    PlanModifiers: []planmodifier.Bool{
        boolplanmodifier.UseStateForUnknown(),
    },
},
