Step 3 — Fix violations by category:

**bodyclose**: Every `httpClient.Do(req)` that does not close resp.Body must be wrapped with `defer resp.Body.Close()`. The auth.go and client.go already do this for the main paths. Check transport.go — the retryTransport.RoundTrip reads/drains bodies on retry but the caller owns the final response body; no fix needed there.

**noctx**: Every `http.Get`, `http.Post`, or `http.NewRequest` without context must become `http.NewRequestWithContext`. Scan all files:
```bash
grep -rn "http\.NewRequest[^W]\|http\.Get\|http\.Post" internal/ --include="*.go"
```
Replace each occurrence with the context-aware variant.

**gosec**: Address any real security issues surfaced. Common false positives to suppress with `//nolint:gosec`:
- `InsecureSkipVerify` in transport.go already has `//nolint:gosec` — verify it's still present after Phase 32 changes
- `G304` (file path in ReadFile for CACertFile) — add `//nolint:gosec` since the path comes from provider config, not user input in an injection-risk sense; OR suppress globally in config

**exhaustive**: Any switch on a typed const enum that lacks a `default` case. Add `default: // unreachable` where needed, or rely on `default-signifies-exhaustive: true` in config to allow explicit default cases.

Step 4 — Iterate until clean:
```bash
golangci-lint run ./...
```
Re-run after each batch of fixes. Aim for zero output.

NOTE: Do NOT suppress violations with broad `//nolint` directives unless the violation is a documented false positive. Fix the code where possible.
