**quota_user_resource.go** — `flashblade_quota_user` resource:

Schema attributes:
- `id`: Computed, UseStateForUnknown — synthetic "file_system_name/uid"
- `file_system_name`: Required, RequiresReplace — the file system this quota applies to
- `uid`: Required, RequiresReplace — the user ID (string, passed as query param). Cannot change after creation.
- `quota`: Required, Int64 — quota limit in bytes. Writable (PATCH).
- `usage`: Computed — current usage in bytes (read-only from API)
- `timeouts`: standard Create/Read/Update/Delete

CRUD methods:
- Create: POST /quotas/users?file_system_names=&uids= with body {quota}. Read back state.
- Read: GET by file_system_name+uid. Drift detection on quota.
- Update: PATCH by file_system_name+uid. Only quota is patchable.
- Delete: DELETE by file_system_name+uid.
- ImportState: Parse "file_system_name/uid" composite ID (SplitN "/" 2). Set file_system_name and uid, readIntoState.

Helpers:
- `readIntoState(ctx, fsName, uid, *model, diags)`
- `mapQuotaUserToModel(qu, *model)` — set ID as "fsName/uid", map all fields

**quota_group_resource.go** — `flashblade_quota_group` resource:

Identical structure to quota_user_resource.go but:
- Replace `uid` with `gid` everywhere
- Replace `User` with `Group` in model mapping
- Use GetQuotaGroup, PostQuotaGroup, PatchQuotaGroup, DeleteQuotaGroup client methods
- Import ID: "file_system_name/gid"

Both resources are simple CRUD — no singleton pattern, no parent/child, no delete guard. Straightforward.
