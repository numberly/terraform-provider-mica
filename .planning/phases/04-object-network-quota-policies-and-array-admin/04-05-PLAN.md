**array_dns_resource.go** — `flashblade_array_dns` singleton resource:

Schema attributes:
- `id`: Computed, UseStateForUnknown
- `domain`: Optional — domain suffix appended by array
- `nameservers`: Optional, List of strings — DNS server IPs
- `services`: Optional, Computed, List of strings — services using this DNS config
- `sources`: Optional, Computed, List of strings — network interfaces for DNS traffic
- `timeouts`: standard Create/Read/Update/Delete

CRUD — SINGLETON LIFECYCLE:
- **Create**: (1) GET /dns to check if config exists. (2) If 404 (no existing config): POST with desired values. If found: PATCH with desired values. (3) Read back full state. This handles both fresh arrays (no DNS) and existing arrays (DNS already configured). See research pitfall 6.
- **Read**: GET /dns. Map to model. Drift detection on domain, nameservers.
- **Update**: PATCH /dns. Build patch comparing plan vs state for domain, nameservers, services, sources.
- **Delete**: PATCH to reset — `nameservers: []` (empty), `domain: ""` (empty string). Log via tflog "DNS config reset to defaults".
- **ImportState**: Accept "default" as import ID. Ignore the ID value — just GET current DNS config. Initialize timeouts null object.

**array_dns_data_source.go** — `flashblade_array_dns` data source:
- No required input (singleton — always reads current config)
- Computed: id, domain, nameservers, services, sources

**array_ntp_resource.go** — `flashblade_array_ntp` singleton resource:

Schema attributes:
- `id`: Computed, UseStateForUnknown
- `ntp_servers`: Required, List of strings — NTP server hostnames/IPs

CRUD — SINGLETON LIFECYCLE (wraps /arrays endpoint):
- **Create**: GET /arrays to read current. PATCH /arrays with {ntp_servers: [...]}. Read back. IMPORTANT: ArrayNtpPatch struct ONLY contains NtpServers field — never send other Array fields (pitfall 5).
- **Read**: GET /arrays. Extract ntp_servers. Drift detection.
- **Update**: PATCH /arrays with {ntp_servers: [new values]}.
- **Delete**: PATCH /arrays with {ntp_servers: []} (empty array). Log "NTP servers cleared".
- **ImportState**: Accept "default". GET /arrays, extract ntp_servers.

**array_ntp_data_source.go** — `flashblade_array_ntp` data source:
- No required input
- Computed: id, ntp_servers

**array_smtp_resource.go** — `flashblade_array_smtp` composite singleton resource:

Per user decision (CONTEXT.md): alert watchers are managed WITHIN the SMTP resource, not as separate resources.

Schema attributes:
- `id`: Computed, UseStateForUnknown
- `relay_host`: Optional — SMTP relay server hostname
- `sender_domain`: Optional — domain appended to sender email
- `encryption_mode`: Optional, Computed — "none", "tls", or "starttls". Default "none".
- `alert_watchers`: Optional, SetNestedAttribute — list of alert watcher objects:
  - `email`: Required (string) — the alert recipient email. This IS the watcher name in the API (pitfall 7).
  - `enabled`: Optional, Computed, Default true
  - `minimum_notification_severity`: Optional, Computed — "info", "warning", "error", "critical". Default "warning".
- `timeouts`: standard

CRUD — COMPOSITE SINGLETON LIFECYCLE:
- **Create**: (1) PATCH /smtp-servers with relay_host, sender_domain, encryption_mode. (2) For each alert_watcher in config: POST /alert-watchers?names=<email> with {minimum_notification_severity}. (3) Read back full state (SMTP + all watchers).
- **Read**: (1) GET /smtp-servers. (2) GET /alert-watchers. (3) Map both into model. Drift detection on relay_host, sender_domain, and watcher list.
- **Update**: (1) PATCH /smtp-servers with changed fields. (2) Diff alert_watchers: compute added/removed/modified watchers. For added: POST. For removed: DELETE. For modified: PATCH (enabled, severity). (3) Read back.
- **Delete**: (1) PATCH /smtp-servers to reset: relay_host="", sender_domain="", encryption_mode="none". (2) DELETE all alert watchers. (3) Log "SMTP config and alert watchers cleared".
- **ImportState**: Accept "default". GET smtp-servers + GET alert-watchers. Build full state.

Alert watcher diff logic in Update:
```
current_emails = set of emails from state
desired_emails = set of emails from plan
to_add = desired - current
to_remove = current - desired
to_update = current & desired (check if enabled or severity changed)
```
