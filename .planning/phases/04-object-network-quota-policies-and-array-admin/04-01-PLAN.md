**models.go additions** — Append these struct families to the end of models.go:

1. **ObjectStoreAccessPolicy** models:
   - `ObjectStoreAccessPolicy` (GET): ID, Name, Description, ARN(string), Enabled(bool), IsLocal(bool), PolicyType(string), Rules([]ObjectStoreAccessPolicyRule), Created(int64), Updated(int64)
   - `ObjectStoreAccessPolicyPost`: Description(string,omitempty)  — name is query param, NOT in body. Rules omitted (created separately via rule endpoint).
   - `ObjectStoreAccessPolicyPatch`: Name(*string,omitempty) — ONLY name is writable on PATCH. NO description (POST-only per research). NO rules (managed via rule endpoint).
   - `ObjectStoreAccessPolicyRule` (GET): Name(string), Effect(string), Actions([]string), Conditions(json.RawMessage), Resources([]string), Policy(*NamedReference)
   - `ObjectStoreAccessPolicyRulePost`: Effect(string), Actions([]string), Conditions(json.RawMessage,omitempty), Resources([]string)
   - `ObjectStoreAccessPolicyRulePatch`: Actions([]string,omitempty), Conditions(json.RawMessage,omitempty), Resources([]string,omitempty) — NO effect (read-only after creation, RequiresReplace in TF schema)

2. **NetworkAccessPolicy** models:
   - `NetworkAccessPolicy` (GET): ID, Name, Enabled(bool), IsLocal(bool), PolicyType(string), Version(string)
   - `NetworkAccessPolicyPatch`: Enabled(*bool,omitempty), Name(*string,omitempty) — NO POST model (singleton, no POST endpoint)
   - `NetworkAccessPolicyRule` (GET): ID, Name(string), Client(string), Effect(string), Index(int), Interfaces([]string), Policy(*NamedReference), PolicyVersion(string)
   - `NetworkAccessPolicyRulePost`: Client(string), Effect(string), Index(int,omitempty), Interfaces([]string)
   - `NetworkAccessPolicyRulePatch`: Client(*string,omitempty), Effect(*string,omitempty), Index(*int,omitempty), Interfaces([]string,omitempty)

3. **Quota** models (per-filesystem user/group quotas — no "quota policy" endpoint exists):
   - `QuotaUser` (GET): FileSystem(*NamedReference), User(*NamedReference), Quota(int64), Usage(int64)
   - `QuotaUserPost`: Quota(int64) — file_system and user specified via query params
   - `QuotaUserPatch`: Quota(*int64,omitempty)
   - `QuotaGroup` (GET): FileSystem(*NamedReference), Group(*NamedReference), Quota(int64), Usage(int64)
   - `QuotaGroupPost`: Quota(int64)
   - `QuotaGroupPatch`: Quota(*int64,omitempty)

4. **Array Admin** models:
   - `ArrayDns` (GET): ID, Name, Domain(string), Nameservers([]string), Services([]string), Sources([]string)
   - `ArrayDnsPost`: Domain(string,omitempty), Nameservers([]string,omitempty), Services([]string,omitempty), Sources([]string,omitempty)
   - `ArrayDnsPatch`: Domain(*string,omitempty), Nameservers(*[]string,omitempty), Services(*[]string,omitempty), Sources(*[]string,omitempty)
   - `ArrayInfo` (GET — partial, NTP-relevant fields only): ID, Name, NtpServers([]string)

   - `AlertWatcherPost`: MinimumNotificationSeverity(string,omitempty) — name (email) is query param
   - `AlertWatcherPatch`: Enabled(*bool,omitempty), MinimumNotificationSeverity(*string,omitempty)
