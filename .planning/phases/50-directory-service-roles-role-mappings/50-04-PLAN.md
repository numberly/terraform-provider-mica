<objective>
Implement the `flashblade_management_access_policy_directory_service_role_membership` Terraform resource — the supplementary attach path for adding a management access policy to an existing directory service role without replacing the role. Follows the proven composite-ID member-resource pattern (qos_policy_member, tls_policy_member, certificate_group_member) with one critical deviation:

**D-05: composite ID puts `role_name` FIRST** (format: `role_name/policy_name`). Rationale: built-in policy names contain both `:` and `/` (e.g. `pure:policy/array_admin`). With `parseCompositeID(id, 2)` using `strings.SplitN(id, "/", 2)`, the role name (simple identifier — no special chars) must come first so the split returns `["role", "pure:policy/array_admin"]` correctly.

Purpose: enables `additional_policies.for_each = toset(policy_names)` idioms and makes post-creation attach/detach ergonomic. Without this resource, adding a policy would force role re-creation (RequiresReplace on DSR).
Output: two new files in `internal/provider/`. Conforms to CONVENTIONS.md §"Resource Implementation" exactly, including all 4 interface assertions (D-00-b).
</objective>
