**# swagger-to-reference**

**## Purpose**
One paragraph: converts OpenAPI 3.0 swagger.json to AI-optimized markdown reference for the FlashBlade provider. Output is consumed by Claude for API exploration and used as base for diff/upgrade skills.

**## When to Use**
- New FlashBlade API version is available (e.g., swagger-2.23.json)
- Regenerating reference after swagger corrections
- Bootstrapping a new API version reference file

**## Prerequisites**
- swagger.json file in project root or known path
- `.claude/skills/_shared/swagger_utils.py` present (Phase 43 shared library)

**## Workflow**

Step 1 — Ask for API version (MANDATORY before running script):

  Before running the script, ask the user:
    "What is the API version string for this swagger file? (e.g., 2.22, 2.23)"
  Do NOT infer from swagger info.version without confirming — the user answer becomes --version.

Step 2 — Run the converter:
```bash
PYTHONPATH=.claude/skills python3 .claude/skills/swagger-to-reference/scripts/parse_swagger.py \
  <path/to/swagger.json> \
  --version <X.XX> \
  --output api_references/<X.XX>.md
```

Step 3 — Verify output:
```bash
# Check header and counts
head -3 api_references/<X.XX>.md

# Confirm no unresolved refs
grep -c "\$ref\|allOf" api_references/<X.XX>.md || echo "0 - PASS"

# Check section presence
grep "^## " api_references/<X.XX>.md
```

Step 4 — Spot-check against swagger:
```bash
python3 -c "import json; d=json.load(open('<swagger.json>')); print(len(d['paths']), 'paths')"
```
The summary line in the output must match this path count.

**## Output Format**
Four sections: header/summary line, Auth, Common Parameters, Endpoints (grouped by tag with H3 per tag), Data Models (compact bold-name field list). See `FLASHBLADE_API.md` in project root as the ground-truth format example.

**## Troubleshooting**
- `ModuleNotFoundError: No module named '_shared'` — ensure `PYTHONPATH=.claude/skills` is set
- Path count mismatch — check the summary line against `len(swagger["paths"])` directly
- `allOf` still in output — verify `resolve_all_of` was called; check `self.resolved_schemas` is populated
