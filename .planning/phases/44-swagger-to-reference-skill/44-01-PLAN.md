**CLI interface (argparse):**
- `swagger_file` positional: path to swagger.json
- `--output` optional: output file path (default: `api_references/{version}.md` relative to swagger_file parent)
- `--version` optional: API version string (e.g. `2.22`); if omitted, extract from swagger_dict `info.version`

**Class `SwaggerConverter`:**

`__init__(self, swagger_path, version=None)`:
- Load swagger.json with `json.load`
- Set `self.version` from `--version` arg or `swagger_dict["info"]["version"]`
- Call `resolve_all_of(swagger_dict)` → `self.resolved_schemas`
- Extract `swagger_dict["paths"]` → `self.paths`

`convert(self) -> str`:
- Build output sections in order: title, summary, auth, common_params, endpoints, data_models
- Return joined string

`_build_title(self) -> str`:
- `# FlashBlade REST API {self.version} — AI-Optimized Reference`

`_build_summary(self) -> str`:
- Count paths: `len(self.paths)`
- Count ops: sum of HTTP method keys across all paths (skip non-method keys like `parameters`)
- Valid HTTP methods: `get, post, put, patch, delete, head, options`
- Format: `Base: \`https://{array}/\` | Version: {self.version} | {N} paths | {M} ops | Auth: \`x-auth-token\` or \`api-token\` header`

`_build_auth(self) -> str`:
- Return the static auth section exactly matching FLASHBLADE_API.md lines 5-17:
  - OAuth2 block (POST /oauth2/1.0/token with full grant_type example)
  - Session login: `POST /api/login` with api-token header → x-auth-token response
  - Session logout: `POST /api/logout`
  - API version: `GET /api/api_version` → {versions: [string]}

`_build_common_params(self) -> str`:
- Collect all query parameters from all path operations
- Deduplicate by name
- Build markdown table: `| \`name\` | type | description truncated to 70 chars... |`
- Sort alphabetically by param name
- Append filter syntax note: `**Filter syntax:** field op value. Ops: =, !=, <, >, <=, >=, and, or, not.`

`_build_endpoints(self) -> str`:
- Group paths by first tag of first operation (or "Other" if no tags)
- Sort tags alphabetically (use title-case display: `active-directory` tag → `Active Directory` heading)
  - Convert tag to title: split on `-`, title-case each word, join with space
- Per tag: `### {Tag Title}` heading
- Per path, sort HTTP methods in canonical order: GET, POST, PUT, PATCH, DELETE
- Per operation line format:
  `- **{METHOD}** \`{full_path_with_version}\``
  If has non-common query params: ` | Params: \`name\`(type), ...`
  If has requestBody: ` | Body: \`field\`(ro? type), ...`
- Extract requestBody fields from `requestBody.content["application/json"].schema`
  - Resolve $ref/allOf using flatten_schema
  - For each property: mark `ro` if `readOnly: true` in the property schema
  - Format: `` `field`(ro type) `` or `` `field`(type) ``
- Non-common params: params not in the common_params set (filter, sort, limit, offset, names, ids, continuation_token, allow_errors, context_names, member_ids, member_names, policy_ids, policy_names, X-Request-ID)

`_build_data_models(self) -> str`:
- Header: `## Data Models (Key Resources)`
- For each schema in `self.resolved_schemas`, sorted alphabetically:
  - flat = flatten_schema(schema, self.resolved_schemas)
  - Per-field format: `` `name`(ro type): description truncated at 50 chars ``
  - Line format: `**SchemaName**: field1 | field2 | field3`
  - Sort fields alphabetically within each schema
  - Truncate description at 50 chars with no ellipsis if fits, otherwise keep first 50 chars (the existing FLASHBLADE_API.md truncates mid-word at ~50 chars — match this behavior)

`_format_type(self, prop_schema: dict) -> str`:
- Return the OpenAPI type string: prop_schema.get("type", "object")
- For arrays: "array", for $ref: resolve to "object" if not further typed

`main()`:
- argparse setup
- Instantiate SwaggerConverter
- Call convert()
- Write to output path (create parent dirs with `pathlib.Path.mkdir(parents=True, exist_ok=True)`)
- Print: `Written: {output_path}` to stderr
- Do NOT prompt for version interactively — the SKILL.md workflow handles the --version prompt
  (The SKILL.md instructions tell Claude to ask the user and pass --version to the CLI)

**Error handling:**
- FileNotFoundError if swagger_file not found → print error to stderr, exit(1)
- json.JSONDecodeError → print error to stderr, exit(1)

**stdlib only:** json, sys, pathlib, argparse, re, collections — no external deps.
