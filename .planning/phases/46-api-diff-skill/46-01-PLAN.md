CLI signature:
```
python3 diff_swagger.py <old.json> <new.json> [--output path] [--format json|markdown] [--discrepancies path]
```

Import shared library:
```python
import sys, os
sys.path.insert(0, os.path.join(os.path.dirname(__file__), "..", "..", "_shared"))
import swagger_utils
```

**Core logic:**

1. Load both swagger files with `json.load`.
2. Build endpoint maps for each: `{normalized_path: {method: operation_dict}}`.
   - Use `swagger_utils.normalize_path(path)` as key — CRITICAL to strip version prefix.
   - Iterate `swagger["paths"]`, then each HTTP method (`get`, `post`, `patch`, `delete`, `put`).
3. Build schema maps: `resolve_all_of(swagger)` for each file.
4. Compute diff:
   - `new_endpoints`: normalized_path+method in new not in old
   - `removed_endpoints`: in old not in new
   - `modified_endpoints`: in both — compare operation `summary`, `parameters` list (names), `requestBody` presence, `responses` keys
   - `new_schemas`: schema name in new not in old
   - `removed_schemas`: in old not in new
   - `modified_schemas`: in both — compare `properties` keys using `flatten_schema`; record `added_fields`, `removed_fields`, `changed_fields`

Each diff item structure:
```json
{
  "normalized_path": "buckets",
  "method": "get",
  "change_type": "new_endpoint",
  "annotation": "needs_verification",
  "details": {}
}
```

`annotation` default: `"needs_verification"`. Override from `--discrepancies` file if provided (see below).

5. Discrepancies file (optional `--discrepancies`): load JSON with structure:
```json
{
  "overrides": [
    {"normalized_path": "buckets", "method": "get", "annotation": "swagger_artifact", "note": "..."}
  ]
}
```
Apply overrides by matching `(normalized_path, method)` pairs after computing diff.

6. Output JSON (default or `--format json`):
```json
{
  "old_version": "<info.version from old swagger>",
  "new_version": "<info.version from new swagger>",
  "summary": {
    "new_endpoints": 5,
    "removed_endpoints": 0,
    "modified_endpoints": 12,
    "new_schemas": 3,
    "removed_schemas": 0,
    "modified_schemas": 8
  },
  "new_endpoints": [...],
  "removed_endpoints": [...],
  "modified_endpoints": [...],
  "new_schemas": [...],
  "removed_schemas": [...],
  "modified_schemas": [...]
}
```

7. Output markdown (`--format markdown`): emit a summary table + one section per category. Each section lists items as a markdown table with columns: Path, Method, Change, Annotation, Details.

8. `--output path`: write result to file; if omitted, print to stdout.

Python 3.10+ stdlib only: `json`, `argparse`, `sys`, `os`, `pathlib`.
