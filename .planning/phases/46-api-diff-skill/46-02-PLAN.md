  <action>
Create `.claude/skills/api-diff/scripts/generate_migration_plan.py`.

CLI signature:
```
python3 generate_migration_plan.py <diff.json> <ROADMAP.md> [--output path] [--format json|markdown]
```

**Core logic:**

1. Load `diff.json` from plan 01 output.
2. Parse ROADMAP.md to extract "Not Implemented" entries:
   - Find the `## Not Implemented` section header.
   - Parse all table rows (lines starting with `|` that are not header or separator lines).
   - Extract `api_section` (col 1) and `status` (last col, strip whitespace).
   - Keep entries where status is `Candidate` or `Deferred`.
   - Build a lookup: normalize `api_section` to lowercase slug for matching
     (e.g., "File System Replica Links" → "file-system-replica-links", "file-systems/replica-links", etc.)
   - Also keep raw name for display.

3. For each `new_endpoint` in diff:
   - Attempt fuzzy cross-reference: does the `normalized_path` contain any word from a roadmap entry's api_section?
   - Strategy: split api_section into words, check if ≥2 words appear in normalized_path (case-insensitive).
   - If match found → tag as `roadmap_gap` with the roadmap entry name.

4. Build migration plan output:

```json
{
  "generated_from": {"old_version": "...", "new_version": "..."},
  "summary": {
    "update_models": 8,
    "new_resources": 3,
    "deprecated": 0,
    "roadmap_gaps": 2
  },
  "update_models": [
    {
      "schema_name": "Bucket",
      "added_fields": ["new_field"],
      "removed_fields": [],
      "annotation": "real_change",
      "action": "Add new_field to BucketPatch and BucketPost structs"
    }
  ],
  "new_resources": [
    {
      "normalized_path": "new-resource",
      "methods": ["get", "post", "patch", "delete"],
      "annotation": "needs_verification",
      "roadmap_status": null,
      "action": "Evaluate for new Terraform resource"
    }
  ],
  "deprecated": [...],
  "roadmap_gaps": [
    {
      "normalized_path": "file-systems/replica-links",
      "roadmap_entry": "File System Replica Links",
      "roadmap_status": "Candidate",
      "annotation": "real_change",
      "action": "New endpoint matches ROADMAP Candidate — schedule implementation"
    }
  ]
}
```

Classification rules:
- `update_models`: items in `modified_schemas` with `annotation != swagger_artifact`
- `new_resources`: items in `new_endpoints` (deduplicated by normalized_path, method=GET as anchor)
  where annotation != `swagger_artifact`
- `deprecated`: items in `removed_endpoints` or `removed_schemas` with annotation != `swagger_artifact`
- `roadmap_gaps`: subset of new_resources that matched a ROADMAP entry

Action strings are generated automatically based on change type — do NOT require LLM, use templates:
- modified_schema: `"Add {', '.join(added_fields)} to {schema_name}Post/Patch structs"`
- new endpoint: `"Evaluate for new Terraform resource flashblade_{snake_path}"`
- roadmap_gap: `"New endpoint matches ROADMAP Candidate — schedule implementation"`

5. Markdown output (`--format markdown`): emit executive summary + one section per category,
   each as a markdown table. Include `## Action Items` at the end listing all non-swagger_artifact items.

6. `--output path`: write to file; if omitted, print to stdout.

Python 3.10+ stdlib only: `json`, `argparse`, `re`, `sys`, `os`, `pathlib`.
