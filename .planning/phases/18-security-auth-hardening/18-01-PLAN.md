Output: Hardened auth.go, client.go, and provider.go with full test coverage of new behavior.
</objective>

  <action>
    **SEC-01 — Sanitize OAuth2 error body (auth.go line 130):**
    In fetchToken(), replace:
    ```go
    return nil, fmt.Errorf("token exchange: HTTP %d: %s", resp.StatusCode, string(body))
    ```
    with a sanitized version that only includes the status code:
    ```go
    return nil, fmt.Errorf("token exchange: unexpected HTTP %d from %s/oauth2/1.0/token", resp.StatusCode, ts.endpoint)
    ```

    **SEC-03 — Add context to fetchToken() (auth.go line 103):**
    Change signature from `fetchToken()` to `fetchToken(ctx context.Context)`.
    Use `http.NewRequestWithContext(ctx, ...)` instead of `http.NewRequest(...)`.
    Update Token() to pass `context.Background()` since oauth2.TokenSource.Token() has no context parameter — add a TODO comment noting this limitation.

    **ERR-04 — Fix LoginWithAPIToken (auth.go lines 26-32):**
    Replace the `http.NewRequest` + nil-check + `WithContext` pattern:
    ```go
    req, err := http.NewRequest(http.MethodPost, loginURL, nil)
    ...
    if ctx != nil {
        req = req.WithContext(ctx)
    }
    ```
    with direct:
    ```go
    req, err := http.NewRequestWithContext(ctx, http.MethodPost, loginURL, nil)
    ```
    Remove the nil ctx guard entirely — callers must provide a valid context.

    **Update tests (auth_test.go):**
    - Add `TestUnit_OAuth2TokenSource_ErrorSanitized`: mock server returns 401 with a body containing `{"error":"invalid_grant","error_description":"token expired"}` — assert error message contains "unexpected HTTP 401" but NOT "invalid_grant" or "token expired".
    - Add `TestUnit_OAuth2TokenSource_ContextCancelled`: create a cancelled context, call a context-aware path, assert context error propagates.
    - Update `TestUnit_LoginWithAPIToken` to pass `context.Background()` instead of `context.TODO()`.
