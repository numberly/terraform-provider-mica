<objective>
Implement the SMB client policy resource family: policy resource (CRUD + import), rule resource (CRUD + import), data source, client methods, mock handler, and unit tests.

Purpose: SMB client policies control SMB client access (client matching, encryption, permissions) at the policy level. This mirrors the existing SMB share policy pattern with key differences: `version` field (read-only), `access_based_enumeration_enabled` field, and rule fields are `client`/`encryption`/`permission` instead of `principal`/`change`/`full_control`/`read`.

Output: 9 files — models in models.go, client CRUD, mock handler, resource, rule resource, data source, provider registration, and 2 test files.

    ID   string `json:"id,omitempty"`
    Name string `json:"name"`
    Enabled bool `json:"enabled"`
    IsLocal bool `json:"is_local,omitempty"`
    PolicyType string `json:"policy_type,omitempty"`
    Version string `json:"version,omitempty"`  // NEW: read-only, Computed+UseStateForUnknown
    AccessBasedEnumerationEnabled bool `json:"access_based_enumeration_enabled,omitempty"` // NEW: Optional bool
    Rules []SmbClientPolicyRuleInPolicy `json:"rules,omitempty"`
