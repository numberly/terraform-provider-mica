## Endpoint section (under "## Endpoints"):
```
### Buckets

- **GET** `/api/2.22/buckets` | Params: `destroyed`(boolean) | Body: `field`(type)
- **POST** `/api/2.22/buckets` | Body: `account`(object), `name`(string)
- **PATCH** `/api/2.22/buckets` | Body: `destroyed`(boolean), `name`(string)
- **DELETE** `/api/2.22/buckets`
```

Key points:
- Tag section header: `### Tag Name` (title-case, may contain spaces, e.g. "Array Connections")
- Endpoint line regex: `^- \*\*(\w+)\*\*\s+` `` `([^`]+)` ``
- Optional suffix after path: ` | Params: ...` and/or ` | Body: ...`
- Body fields format: `` `fieldname`(ro type) `` or `` `fieldname`(type) ``

## Data Models section (under "## Data Models (Key Resources)"):
```
**BucketPost**: `account`(object): The account ... | `hard_limit_enabled`(boolean): ... | `name`(string): ...
**BucketPatch**: `destroyed`(boolean): ... | `hard_limit_enabled`(boolean): ...
```

Key points:
- Schema line regex: `^\*\*([A-Za-z0-9_]+)\*\*:\s*(.+)`
- Fields are pipe-separated: `` `fieldname`(ro type): description ``
- `ro ` prefix before type means readOnly=true
- No explicit `required` info in the markdown (not emitted by parse_swagger.py â€” treat as not available, omit from output)
- Field regex per pipe segment: `` `(\w+)`\((ro )?([^)]+)\):\s*(.*)  ``
