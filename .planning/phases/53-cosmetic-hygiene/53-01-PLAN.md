### 1. Edit `CONVENTIONS.md` §Pointer rules (line 35 onward)

After the existing line 43-47 bullet describing `PATCH struct` pointer rules (ends with the `**NamedReference` bullet), insert a new paragraph:

```markdown
#### Exception — "always send" list fields

Some FlashBlade API endpoints treat an absent list key and an empty list as distinct: the only way to clear the list is to send `"field": []` in the JSON body. For these endpoints, the PATCH struct field MUST be a plain `[]T` **without** `omitempty` — the normal `*[]T`+`omitempty` pattern would wrongly omit the field when the caller passes an empty slice.

Known exceptions (do not migrate):

- `NetworkInterfacePatch.Services` (`internal/client/models_network.go`)
- `NetworkInterfacePatch.AttachedServers` (`internal/client/models_network.go`)

When adding a new PATCH struct, default to `*[]T` with `omitempty`. Only fall back to plain `[]T` when the API is verified to reject absent keys as "no change" semantics that differ from an empty list.
```

Keep the rest of the file unchanged.
