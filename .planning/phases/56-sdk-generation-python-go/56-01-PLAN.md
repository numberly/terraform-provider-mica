# Phase 56, Plan 01: Schema Artifacts and Makefile Targets

## Objective

Ensure all committed schema artifacts are correct and complete, add `generate_python` and `generate_go` Makefile targets, and establish the schema drift gate. No NodeJS/C#/Java targets per SDK-04.

## Context

The Pulumi bridge uses three committed schema artifacts:
- `schema.json` — already generated by `make tfgen`
- `bridge-metadata.json` — already generated by `make tfgen`
- `schema-embed.json` — **MISSING**; required for SDK generation but not currently produced

The `tfgen` target in `pulumi/Makefile` runs `$(TFGEN_BIN) schema --out $(SCHEMA_DIR)` which emits `schema.json` and `bridge-metadata.json`. It does NOT emit `schema-embed.json`. The `schema-embed.json` is typically a minified single-line version of `schema.json` used for embedding.

The Makefile currently defines: `tfgen_build`, `tfgen`, `provider`, `test`, `clean`. It does NOT define `generate_python` or `generate_go`.

<id>02</id>
<title>Add generate_python Makefile target</title>
<read_first>
- pulumi/Makefile
- pulumi/provider/cmd/pulumi-resource-flashblade/schema.json
</read_first>
<action>
Add a `generate_python` target to `pulumi/Makefile` that generates the Python SDK using the Pulumi codegen tools. The target must:

1. Ensure `tfgen` has run (schema artifacts exist)
2. Create the output directory `pulumi/sdk/python/`
3. Run Python SDK generation using `pulumi-tfgen` or `pulumi codegen` — since the project uses the PF bridge, use the standard bridge SDK generation approach:

```makefile
.PHONY: generate_python

TFGEN_BIN := $(PROVIDER_DIR)/cmd/pulumi-tfgen-flashblade/pulumi-tfgen-flashblade
SCHEMA_DIR := $(PROVIDER_DIR)/cmd/pulumi-resource-flashblade
PYTHON_SDK_DIR := sdk/python

generate_python: tfgen
	@echo "Generating Python SDK..."
	@rm -rf $(PYTHON_SDK_DIR)
	@mkdir -p $(PYTHON_SDK_DIR)
	$(TFGEN_BIN) python --out $(PYTHON_SDK_DIR) $(SCHEMA_DIR)/schema.json
	@# The bridge tfgen python subcommand emits SDK files into the output directory
	@test -f $(PYTHON_SDK_DIR)/setup.py || test -f $(PYTHON_SDK_DIR)/pyproject.toml || (echo "Python SDK generation failed: no setup.py or pyproject.toml" && exit 1)
	@echo "Python SDK generated in $(PYTHON_SDK_DIR)"
```

**Note**: If `pulumi-tfgen-flashblade` does not support a `python` subcommand directly, use the Pulumi codegen Python generator:
```makefile
generate_python: tfgen
	@rm -rf $(PYTHON_SDK_DIR)
	@mkdir -p $(PYTHON_SDK_DIR)
	pulumi plugin install resource flashblade $(VERSION) --server $(PLUGIN_SERVER) 2>/dev/null || true
	pulumi gen-sdk python --out $(PYTHON_SDK_DIR) $(SCHEMA_DIR)/schema.json
```

The executor must verify which approach works with the installed bridge version and implement the correct one. The acceptance criteria must pass regardless of the specific command used.
</action>
<acceptance_criteria>
- `pulumi/Makefile` contains a `generate_python` target
- The target depends on `tfgen` (ensures schema artifacts exist first)
- The target creates files in `pulumi/sdk/python/`
- `make generate_python` exits 0
- After running, `pulumi/sdk/python/` contains either `setup.py` or `pyproject.toml`
- After running, `pulumi/sdk/python/` contains a `pulumi_flashblade/` package directory or module
</acceptance_criteria>
</task>

<action>
Add a `generate_go` target to `pulumi/Makefile` that generates the Go SDK. The target must:

1. Ensure `tfgen` has run (schema artifacts exist)
2. Generate Go SDK files into `pulumi/sdk/go/`
3. Preserve the existing `go.mod` module path: `github.com/numberly/opentofu-provider-flashblade/pulumi/sdk/go`

```makefile
.PHONY: generate_go

GO_SDK_DIR := sdk/go

generate_go: tfgen
	@echo "Generating Go SDK..."
	@rm -rf $(GO_SDK_DIR)/*.go $(GO_SDK_DIR)/*/
	@# Preserve go.mod
	@cp $(GO_SDK_DIR)/go.mod $(GO_SDK_DIR)/go.mod.bak
	$(TFGEN_BIN) go --out $(GO_SDK_DIR) $(SCHEMA_DIR)/schema.json
	@# Alternative if tfgen go subcommand unavailable:
	@# pulumi gen-sdk go --out $(GO_SDK_DIR) $(SCHEMA_DIR)/schema.json
	@mv $(GO_SDK_DIR)/go.mod.bak $(GO_SDK_DIR)/go.mod
	@test -f $(GO_SDK_DIR)/flashblade/pulumiTypes.go || test -d $(GO_SDK_DIR)/flashblade || (echo "Go SDK generation failed" && exit 1)
	@echo "Go SDK generated in $(GO_SDK_DIR)"
```

The executor must verify the correct subcommand/approach and adjust accordingly. The key requirement is that generated Go code lands in `pulumi/sdk/go/` under the correct module path.
</action>
