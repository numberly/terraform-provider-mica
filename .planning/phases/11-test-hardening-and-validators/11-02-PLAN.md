Output: Shared query_params.go helper + 4 hardened mock handlers.

  <action>
Create `internal/testmock/handlers/query_params.go` with a helper function:

```go
// ValidateQueryParams checks that all query parameters in the request are in the allowed set.
// If an unknown param is found, it writes a 400 JSON error response and returns false.
// The caller should return immediately when false is returned.
//
// allowedParams is the set of param names valid for this endpoint+method combination.
// Always include framework params that the real API accepts globally: "continuation_token", "limit", "offset", "sort", "filter", "total_item_count".
func ValidateQueryParams(w http.ResponseWriter, r *http.Request, allowedParams []string) bool
```

Implementation:
- Build a `map[string]bool` from `allowedParams` plus the global params listed above.
- Iterate `r.URL.Query()` keys. If any key is not in the map, write a 400 JSON response: `{"errors":[{"message":"unknown query parameter: <name>"}]}` and return false.
- Return true if all params are valid.

Also create a helper for mandatory params:

```go
// RequireQueryParam checks that the given param is present and non-empty.
// Writes 400 and returns "" if missing.
func RequireQueryParam(w http.ResponseWriter, r *http.Request, param string) (string, bool)
```

Create `internal/testmock/handlers/query_params_test.go` with unit tests:
- ValidateQueryParams: accepts request with known params, rejects request with unknown param "foo"
- RequireQueryParam: returns value when present, returns false when missing
- Use `httptest.NewRecorder` and `http.NewRequest` for test setup.
