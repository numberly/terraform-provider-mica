**CRITICAL: This resource is architecturally different from NFS/SMB rule resources.**

There is NO dedicated API endpoint for snapshot rules. All operations go through PATCH on the parent policy.

Schema:
- `id` — Computed, UseStateForUnknown. Since snapshot rules don't have a UUID `id` from the API, use a synthetic ID: `{policy_name}/{rule_name}` as the stored id.
- `policy_name` — Required, RequiresReplace
- `name` — Computed, UseStateForUnknown (server-assigned rule identifier within the policy)
- `at` — Optional (epoch ms — schedule: run at this time)
- `every` — Optional (interval ms — schedule: run every N ms)
- `keep_for` — Optional (retention ms — keep snapshots for this long)
- `suffix` — Optional
- `client_name` — Optional
- `timeouts` — standard CRUD

NOTE: These schedule fields are best-effort based on FlashBlade convention. The API reference does not fully enumerate them. If compilation or tests reveal different field names, adjust accordingly. The mock handler should be flexible (store raw JSON for rule bodies).

**Create**:
1. Read plan, extract policy_name and rule fields
2. Build SnapshotPolicyRulePost from fields
3. Call AddSnapshotPolicyRule(ctx, policyName, rulePost)
4. Response is the updated SnapshotPolicy — find the newly added rule (last in Rules array, or by matching fields)
5. Store rule.Name (server-assigned) in state for future operations

**Read**:
1. Call GetSnapshotPolicy(ctx, policyName)
2. Find rule in policy.Rules by stored name
3. If not found, RemoveResource (deleted externally)
4. Map fields to state

**Update**:
1. Call ReplaceSnapshotPolicyRule(ctx, policyName, oldRuleName, newRulePost)
2. Response is updated policy — find the new rule (may have new server-assigned name)
3. Update state with new name and all fields

**Delete**:
1. Call RemoveSnapshotPolicyRule(ctx, policyName, ruleName)

**ImportState**:
1. Parse `strings.SplitN(req.ID, "/", 2)` — format: `policy_name/rule_index`
2. Validate parts[1] is a valid integer (index = position in rules array)
3. Call GetSnapshotPolicyRuleByIndex(ctx, policyName, index) to get the rule
4. Set policy_name and name (server-assigned) in state
5. Read full state
