Follow `object_store_account_resource.go` pattern exactly. Key differences:

Schema attributes:
- `id` — Computed, UseStateForUnknown
- `name` — Required (NOT RequiresReplace — rename is in-place via PATCH per user decision)
- `enabled` — Optional, Computed, default true via `booldefault.StaticBool(true)`
- `is_local` — Computed, UseStateForUnknown
- `policy_type` — Computed, UseStateForUnknown
- `version` — Computed (changes on each update, do NOT use UseStateForUnknown)
- `timeouts` — standard CRUD timeouts

Create: POST with name as query param, body has `enabled` only.
Read: GET by name, map to model, drift detection on `enabled` field.
Update: build NfsExportPolicyPatch with pointer fields. If name changed, set patch.Name. If enabled changed, set patch.Enabled.
Delete: **GUARD** — call ListNfsExportPolicyMembers first. If members > 0, AddError("Policy is in use, detach first."). Otherwise DELETE.
ImportState: parse name from req.ID, read into state (same pattern as object_store_account).

Read-only. Schema mirrors resource minus timeouts. Has `name` as Required filter. Calls GetNfsExportPolicy, maps to model.

**NFS Export Policy Rule Resource** (`nfs_export_policy_rule_resource.go`):

Schema attributes:
- `id` — Computed, UseStateForUnknown (UUID, server-assigned)
- `policy_name` — Required, ForceNew (RequiresReplace — rule belongs to one policy, moving is not supported)
- `name` — Computed, UseStateForUnknown (server-assigned rule identifier — used for API targeting)
- `index` — Computed (server-assigned ordering — used for import only, NOT UseStateForUnknown because it can change)
- `policy_version` — Computed
- `access` — Optional, Computed (default: "root-squash")
- `client` — Optional, Computed (default: "*")
- `permission` — Optional, Computed (default: "rw")
- `anonuid` — Optional, Computed
- `anongid` — Optional, Computed
- `atime` — Optional, Computed
- `fileid_32bit` — Optional, Computed
- `secure` — Optional, Computed
- `security` — Optional, Computed, List of strings
- `required_transport_security` — Optional, Computed
- `timeouts` — standard CRUD

Create:
1. Read plan, extract policy_name and rule fields
2. Build NfsExportPolicyRulePost with all writable fields
3. Call PostNfsExportPolicyRule(ctx, policyName, body)
4. Read back the created rule by name (from response) and set full state

Read:
1. Get rule by name from state: GetNfsExportPolicyRuleByName(ctx, policyName, ruleName)
2. If not found, RemoveResource (deleted externally)
3. Map all fields to state, drift detection on mutable fields

Update:
1. Compare plan vs state for all mutable fields
2. Build NfsExportPolicyRulePatch with pointer fields for changed values
3. Call PatchNfsExportPolicyRule(ctx, policyName, ruleName, patch)
4. Read back and set state

Delete:
1. Call DeleteNfsExportPolicyRule(ctx, policyName, ruleName)

ImportState:
1. Parse composite ID with `strings.SplitN(req.ID, "/", 2)` — format: `policy_name/rule_index`
2. Validate parts[1] is a valid integer (rule index)
3. Call GetNfsExportPolicyRuleByIndex(ctx, policyName, index) to find the rule
4. Set policy_name and name (server-assigned) in state
5. Read full state via readIntoState
