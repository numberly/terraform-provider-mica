**Step 3 ‚Äî Create NFS mock handler** in `internal/testmock/handlers/nfs_export_policies.go`:

Follow the `object_store_accounts.go` handler pattern. The store needs two collections:
- `policies map[string]*client.NfsExportPolicy` (by name)
- `rules map[string]map[string]*client.NfsExportPolicyRule` (policyName -> ruleName -> rule)
- `nextRuleIndex map[string]int` (policyName -> next index to assign)

Register two endpoint paths:
- `/api/2.22/nfs-export-policies` ‚Äî dispatches GET/POST/PATCH/DELETE for policies
- `/api/2.22/nfs-export-policies/rules` ‚Äî dispatches GET/POST/PATCH/DELETE for rules

Policy handler behavior:
- POST: create policy with UUID id, store by name. Body fields: `enabled` (default true if absent).
- GET: filter by `?names=` or return all. When returning a single policy, populate its `Rules` field from the rules store (sorted by Index).
- PATCH: raw map decode for true PATCH semantics. Support `name` rename (update map keys). Support `enabled` update.
- DELETE: remove from store. (The resource-level guard checks membership ‚Äî the mock doesn't need to enforce this, but optionally can.)

Rule handler behavior:
- POST: parse `?policy_names=`, create rule with UUID id, auto-assign `name` (e.g., "rule-{uuid-short}"), auto-assign `index` from nextRuleIndex counter. Store in `rules[policyName][ruleName]`.
- GET: filter by `?policy_names=` and optionally `?names=`. Return matching rules.
- PATCH: raw map decode, update fields. Support `index` change for repositioning (move in the index ordering).
- DELETE: remove by `?names=` and `?policy_names=`.

Export `RegisterNfsExportPolicyHandlers(mux *http.ServeMux) *nfsExportPolicyStore` ‚Äî returns store pointer for cross-reference by resource tests.
