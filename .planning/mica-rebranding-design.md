# Mica Rebranding — Design Spec

**Date**: 2026-04-28
**Status**: Approved (brainstorming complete, awaiting implementation plan)
**Target tag**: `v2.22.4`

## Context

Open-sourcing the project requires Pure Storage trademark compliance per
guidelines received from Eugenio Grosso (Pure Storage) on 2026-04-28:

- Cannot use `Pure`, `Pure Storage`, `Everpure`, `FlashBlade`, `Purity`, etc.
  in product/solution name, repo name, or logo.
- Cannot use Pure Storage "P" logo or look-alike iconography.
- Cannot imply official status ("official", "certified", "Pure-approved").
- CAN use trademarks as nominative descriptive references in docs/taglines
  with `™`/`®` symbols and "not affiliated" disclaimer.
- Borderline patterns (e.g. "X for FlashBlade") require clearance from
  `trademarks@purestorage.com`.

## Decision: rename to `mica`

`mica` chosen for:

- **Trademark safety**: dictionary word, geological mineral. Zero
  trademark collision with Pure Storage.
- **Subtle wordplay**: mica is a mineral that cleaves into stacks of thin
  lamellae (the geological/mineralogical term for thin blade-like sheets).
  Subtle reference to FlashBlade's stacked-blade architecture without
  using any restricted term.
- **Form**: 4 letters, pronounceable in FR/EN, memorable.
- **Availability**: no significant OSS collision on `terraform-provider-mica`
  on GitHub, Terraform Registry, OpenTofu Registry.

## Goals

1. Comply with Pure Storage trademark guidelines for open-source release.
2. Maintain HCL compatibility for existing internal Numberly users
   (resource prefix unchanged).
3. Switch license to GPL v3 for strong copyleft protection.
4. Enable publication to Terraform Registry + OpenTofu Registry.
5. Tag the rebranded release as `v2.22.4` (continuation of FlashBlade
   API version tracking scheme).

## Non-goals

- Renaming internal Go identifiers (`FlashBladeClient`, `models_storage.go`,
  etc.) — they are internal, never user-facing, and act as descriptive
  references to the target system.
- Renaming the `flashblade_*` resource/data source prefix — kept as a
  nominative descriptive reference, following the Terraform community
  precedent (`aws_*`, `google_*`, `vsphere_*`).
- Creating a logo at this milestone (deferred).
- Migrating off `terraform-provider-*` naming convention to
  `opentofu-provider-*` (rejected: would lose Terraform Registry publication
  and ecosystem compatibility, with zero license benefit).

## Architecture of changes

### 1. Identity layer (changes)

| Element | Before | After |
|---|---|---|
| GitHub repo | `numberly/opentofu-provider-flashblade` | `numberly/terraform-provider-mica` |
| Go module | `github.com/numberly/opentofu-provider-flashblade` | `github.com/numberly/terraform-provider-mica` |
| Binary name | `terraform-provider-flashblade` | `terraform-provider-mica` |
| Provider Address | `registry.terraform.io/numberly/flashblade` | `registry.terraform.io/numberly/mica` |
| Registry slug | `numberly/flashblade` | `numberly/mica` |
| HCL `required_providers` source | `numberly/flashblade` | `numberly/mica` |
| Default HCL local name | `flashblade` | `mica` (alias to `flashblade` allowed) |
| Pulumi resource prefix | `flashblade:...` | `mica:...` |
| `goreleaser` `project_name` | (current) | `terraform-provider-mica` |

### 2. User-facing HCL (unchanged)

- Resources: `flashblade_bucket`, `flashblade_target`, `flashblade_filesystem`, etc.
- Data sources: `flashblade_*` (idem).
- Rationale: descriptive nominative reference, allowed by guidelines.
  Same pattern as `aws_*` (HashiCorp ≠ Amazon), `google_*` (HashiCorp ≠ Google),
  `vsphere_*` (HashiCorp ≠ VMware).

User HCL alias to keep `provider "flashblade" {}` syntax:
```hcl
terraform {
  required_providers {
    flashblade = {
      source = "numberly/mica"
      version = "~> 2.22.4"
    }
  }
}

provider "flashblade" { ... }

resource "flashblade_bucket" "example" { ... }
```

### 3. Internal Go (unchanged)

- Package names: `client`, `provider`, `testmock` (already neutral).
- Type names: `FlashBladeClient`, `FlashBladeClientConfig`, etc.
  (internal, descriptive of target system).
- File names under `internal/`: unchanged (descriptive).
- Mock URLs and handlers: unchanged.

### 4. Documentation rebrand

- `README.md` retitled `# Mica — Terraform provider for Pure Storage FlashBlade®`.
- Tagline: *"Mica is an open-source Terraform/OpenTofu provider for Pure
  Storage FlashBlade® scale-out storage. Not affiliated with or endorsed by
  Pure Storage, Inc."*
- Trademark notice section in `README.md`:
  > Pure Storage®, FlashBlade®, and Purity® are trademarks of Pure Storage,
  > Inc. This project is independent and not affiliated with, endorsed by,
  > or sponsored by Pure Storage, Inc.
- `™`/`®` symbols on first mention in each documentation file.
- `docs/` regenerated by `make docs` after schema metadata change.
- Examples HCL under `examples/` reorganized: `examples/resources/flashblade_*/`
  unchanged (since prefix is kept), only example `terraform { required_providers }`
  blocks updated to `numberly/mica`.

### 5. License — GPL v3

- Replace any existing license declaration with GPL v3.
- Add `LICENSE` (full GPL v3 text) at repo root.
- Add `NOTICE` listing Pure Storage trademarks and disclaiming affiliation.
- Compatibility notes:
  - `terraform-plugin-framework` (MPL 2.0): compatible with GPL v3 via
    MPL §3.3 secondary licensing clause (FSF + Mozilla position).
  - Provider distributed as binary invoked via gRPC IPC: GPL copyleft does
    not propagate to users' HCL or Terraform CLI itself (IPC barrier =
    license barrier, same as Linux kernel ↔ userspace).
  - Strong copyleft prevents scenario A (Pure forks, closes source,
    re-brands as official Pure provider): any fork must remain GPL.

### 6. Versioning — `v2.22.4`

- Continuation of existing version scheme tracking FlashBlade API version
  (`major.minor` = API version, `patch` = provider iteration).
- CHANGELOG entry documents:
  - Repo rename `opentofu-provider-flashblade` → `terraform-provider-mica`.
  - Module path change.
  - Registry source change `numberly/flashblade` → `numberly/mica`.
  - License change (GPL v3).
  - Resource/data source names unchanged.
- Internal Numberly consumers must update `required_providers` blocks.

### 7. Pulumi bridge

- `pulumi/provider/`: update Go module imports.
- Provider name in bridge config: `mica`.
- Schema regen: all Pulumi resource types from `flashblade:bucket:Bucket`
  to `mica:bucket:Bucket`.
- SDK regen for Go (and any other languages currently published).
- Bridge tests (`TestProviderInfo_*`) updated.

### 8. CI/CD

- `.goreleaser.yml`, `.goreleaser.pulumi.yml`: `project_name` and `binary` updated.
- `.github/workflows/*.yml`: artifact names, release names, badge URLs.
- `terraform-registry-manifest.json`: validate format compatibility (generic, no rename needed).

## Implementation phases

(Details consolidated from migration plan; numbered for execution order.)

1. **Foundations** — module path, binary name, `goreleaser`.
2. **Provider Address** — `main.go`, `internal/provider/provider.go` `Metadata`.
3. **Documentation regen** — README, tagline, disclaimer, ™/®, `make docs`.
4. **License switch** — `LICENSE` (GPL v3 full), `NOTICE`.
5. **Pulumi bridge alignment** — module paths, provider name, schema regen.
6. **CI/CD updates** — goreleaser, GitHub Actions.
7. **GitHub repo rename** — preserves redirect, update local remote.
8. **Validation** — `make build`, `make test`, `make lint`, `make docs`,
   E2E manual test on real array (or mock).
9. **Tag & release** — `v2.22.4` (publication handled separately by user).

Skipped:
- Phase 4 of original plan (rename internal Go identifiers): out of scope.
- Phase 10 of original plan (publication/announcement): handled by user
  outside this implementation work.

## Validation criteria

- `make lint` clean.
- `make test` green, count ≥ 818 baseline.
- `make build` produces binary `terraform-provider-mica`.
- `make docs` regen produces no manual diffs.
- `terraform providers schema -json` shows provider name `mica` and
  resources still named `flashblade_*`.
- Pulumi bridge: `cd pulumi/provider && go build ./... && go test ./...` pass.
- E2E: `terraform init && terraform plan && terraform apply && terraform destroy`
  on a `flashblade_target` resource using `source = "numberly/mica"`.

## Risks

| Risk | Mitigation |
|---|---|
| GPL v3 adoption friction (corporate legal teams blacklist GPL) | Accepted; rationale documented in README; users invoke via Terraform CLI (IPC barrier, no contamination of their code). |
| Trademark gray zone on `flashblade_*` resource prefix | Mitigated by AWS/google/vsphere precedent; Eugenio's email confirms nominative descriptive use is allowed. |
| Numberly internal consumers need to update `source = "numberly/mica"` | Tracked in CHANGELOG; user (Guillaume) handles internal communication. |
| GitHub repo rename breaks existing clones | GitHub maintains HTTP redirect; `git remote set-url` documented in README. |
